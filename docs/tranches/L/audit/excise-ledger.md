## §4 — The 4 architectural transpositions (smell → win)

1. **Branded nominal types retire the 2 `as any` ObjectId casts** (#13, #17). `SessionToken`/`UserSlug` brand `Session._id`/`User._id`; `db.collection<Session>/<User>()` then sees a structurally-compatible filter. **Win**: the cast is RETIRED at the one-time model-boundary assertion, not relocated; self-documenting; drops `as any` to 0. Land atomic with the DI-bypass fix.
2. **`status` 4-state → `(visibility, tier)` filter** (#6–#12). Transpose the query-filter semantics into a `(visibility, tier)` filter, then delete bottom-up (writes → filter → projection → envelope → interface). **Win**: collapses two parallel state representations into one canonical source (KISS + DRY); the list API becomes self-describing (`visibility=public&tier=featured`); retires the I.W4-deferred consumer migration 4 audits flagged.
3. **Route→service ownership/ETag helper** (#16). Lift `findBySlug` reads into `services/palette/ownership.ts`. **Win**: restores routes→services→repositories (never routes→repositories); the ownership/ETag logic becomes unit-testable + reusable (DRY).
4. **Ad-hoc envelopes → typed `ApiError`** (#15). Throw typed subclasses; `onError→toResponseEnvelope` owns every shape. **Win**: one error contract, zero special cases; every failure path greppable as a thrown `ApiError`; consistent problem+json with stable codes. Preserve the 503/401/403/400 codes (no client-observed status drift).

## §5 — Enforcement note

The grep-based `proof:*`-script codification idiom is **retired as overfit**. These invariants are enforced **structurally** — branded types + `tsc` (`strict`/`noImplicitAny`) + `eslint` (`no-explicit-any`) + the excision itself (deleting the field from `models.ts` makes `tsc` reject any surviving read) — and **verified at close by a human-run review/grep**, not a committed script. No `scripts/proof-api-*.mjs` is authored.
