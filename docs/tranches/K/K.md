## §1 — Thesis

**The arc's center of gravity has been the library; its blind spot has been the boundary.** A→J drove `as any` to 0, `as unknown as` to 2, decomposed every god module, hardened a band of structural invariants, closed cascade-correctness, and shipped CRUD-CONTRACT v2. (The grep-based `proof:*` codification idiom those invariants once rode is now retired as overfit — `audit/path-forward-2026-06-02.md §3` — the invariants survive structurally; the scripts are deleted.) That is a hardened library. But the *relationship between value.js and glass-ui* — named in the very first mandate ("use glass-ui for ALL styling and components"; "the AND — gaps in value.js AND glass-ui"; "playwright the blob+aurora and abstract them idiomatically") — was never consummated. It was deferred, tranche after tranche, as "glass-ui peer-authorship-blocked," because no value.js tranche was empowered to author glass-ui source. **K removes that block by being a paired-authorship cohort tranche.**

The empirical validation (W0, dimension 6) is the thesis in evidence. The relationship is not merely incomplete — it is *actively broken at HEAD*:

- **Build-state coupling.** value.js's `vue-tsc` resolves `@mkbabb/glass-ui` to `dist/glass-ui.js` via the package `types` field; when glass-ui's `dist/` is absent (it is — glass-ui is mid-development), 75 cascading `TS7016` errors fire. Meanwhile Vite *dev* resolves glass-ui from *source* (contract-v2). **Two resolvers, two truths.** The TypeScript half of contract-v2 was never completed.
- **Production-API coupling.** The demo's color-names client fetches `https://api.color.babb.dev/colors/approved` unconditionally; the deployed CORS policy allows `https://color.babb.dev`, not the `api.` subdomain and not `localhost`. The fetch is refused, a CORS console error fires, and because `setupEnvNoise` filters only 4xx/5xx *text* (not CORS-preflight-class messages), the `zero console errors` assertion fails across nearly the entire smoke suite. The e2e gate the docs call "green" is RED.
- **Math duplication.** glass-ui's `aurora/composables/color.ts` re-implements value.js's Ottosson OKLab/OKLCh/sRGB matrices byte-for-byte (same `0.4122214708…` / `+4.0767416621…` coefficients). value.js *is* the color library; glass-ui should not own a second copy.
- **Stranded primitive.** goo-blob — a self-contained WebGL2 metaball engine with *zero* value.js coupling — sits in the demo, not glass-ui, even though glass-ui already hosts its sibling (aurora) and has no blob primitive.

The naïve fix to the duplication — "make glass-ui import value.js" — is exactly the cycle the user flagged, *if* one forgets that value.js the **library** and value.js the **demo** are different graph nodes. The library imports glass-ui *never*; the demo imports glass-ui; glass-ui importing the value.js **library** closes **no** loop. K codifies this as **inv-K-1** so no one can "fix" the duplication the wrong way. The whole topology is already acyclic by construction the moment the library/demo distinction is held precisely — K's job is to *activate the dormant correct edge* (`glass-ui → value.js-lib`) and *forbid the wrong one* (`value.js-lib → glass-ui`).

Around that spine, K discharges the two oldest mandates (aurora-derive, blob-extirpation) — now unblocked because K co-authors the glass-ui side — and brings the demo to the modern-web posture glass-ui *already passes*: the demo is one tranche of *consumption-alignment* away from the same Baseline-forward grade. The unifying move throughout: **the demo should consume glass-ui's substrate, glass-ui should consume value.js's color core, and value.js's library should consume nothing of either.** One direction, no cycle, one canonical core, zero legacy.

## §2 — Invariants (new in K)

K inherits all prior value.js invariants (A through J: the `as any`/`as unknown as` budgets, no-`@deprecated`, no-`@ts-ignore`, no-`:deep()`, no-bare-builtins, cascade-correctness, the demo ≤400-LoC god-module cap, inv-I-1 cross-repo source boundary — **amended for glass-ui by this tranche, see below**). K adds **five new invariants by name**. Each is enforced **structurally** — by branded/nominal types + `tsc`/`vue-tsc`, by `eslint`, by the *excision itself* (a deleted field cannot regress), and by close-time review (grep-checks a human runs against the diff at the 7-lane close). **The grep-based `proof:*` codification idiom (formerly G4/H4) is RETIRED as overfit** (`audit/path-forward-2026-06-02.md §3`): no K wave authors or proposes any committed `proof:*` script. Each invariant below names its structural enforcement, not a script.

- **inv-K-1 — Acyclic cross-repo color topology.** The value.js **library** (`src/`) imports glass-ui **never**. glass-ui may import the value.js **library** (the canonical color core) as a peerDependency. The value.js **demo** imports glass-ui. The DAG is exactly: `glass-ui → value.js(lib)` and `value.js(demo) → glass-ui`; no edge ever points `value.js(lib) → glass-ui`. **Structural enforcement** (two parts — source-text alone is insufficient because the current single `tsconfig.json` `include: ["src/", "demo/"]` compiles library + demo in one program, so demo's glass-ui imports sit in the *same* TS program as `src/`): (1) the **published library type-graph is separated** — K.W2 splits the unified config into `tsconfig.lib.json` (`src/` only — the published-library graph, glass-ui-free *by construction*: it cannot import glass-ui because glass-ui is not in its program) and `tsconfig.demo.json` (`demo/`, resolves glass-ui from source); the binding fact is the library build (`tsconfig.lib`) typechecks + emits dts **with glass-ui's `dist/` deleted**; (2) a close-time `grep -r "glass-ui" src/` returns zero (human-run review check, not a committed script). Program-graph isolation (the split) is the load-bearing proof; the grep is the close-time backstop.
- **inv-K-2 — Single canonical color core.** No OKLab/OKLCh/sRGB transfer/matrix/gamut math is implemented outside value.js `src/units/color/`. glass-ui's aurora (and the lifted goo-blob) consume value.js's color API. **Structural enforcement**: an equivalence *test* (vitest) asserting glass-ui's derived OKLCh→linear stops agree with value.js's path to 1e-6, plus the *deletion itself* (`srgbToOKLab`/`oklabToLinearRgb` removed from glass-ui — a deleted symbol cannot duplicate; a close-time grep confirms it is defined exactly once across both repos, in value.js).
- **inv-K-3 — glass-ui-first primitives, color-agnostic by default.** Generic visual/interaction primitives live in glass-ui, parameterized by injected app concerns (color resolver, mood/config sources); the demo holds only *product behavior* (the affective FSM, triggers, wiring). New shared primitives are added to glass-ui, never to `demo/@/components/ui/`. **Critical design point** (the latent-dep trap): the lifted **blob primitive takes a *required* injected color resolver and ships NO `value.js` default** — the primitive only ever needs RGB, so baking `parseCSSColor` as its default would make value.js a hard runtime dep of the glass-ui blob for *every* external consumer (a needless coupling, even though inv-K-1 would permit it). The value.js *demo* injects `parseCSSColor`; an external glass-ui consumer injects their own. (Contrast aurora's *derive* path, which genuinely needs OKLab and so consumes value.js by design under inv-K-2 — the blob does not.) (Codifies `feedback_glass_ui_first_class.md`.) **Structural enforcement** (the excision itself + close-time review): (1) the deletions — `git` shows `demo/@/components/custom/{goo-blob,watercolor-dot}/` removed; (2) every `demo/@/components/ui/*/index.ts` is a bare `export … from "@mkbabb/glass-ui…"` re-export (a K.W1-authored whitelist names the permitted shims; a close-time grep confirms); (3) the blob primitive's resolver parameter has no value.js default (a close-time grep of the glass-ui blob source for `parseCSSColor` → zero).
- **inv-K-4 — Contract-v2 resolution completeness.** Both Vite (runtime/dev) **and** TypeScript (`vue-tsc`) resolve cross-repo workspace deps from **source**, not built `dist/`. value.js's typecheck must not depend on glass-ui's build state. **Structural enforcement**: the `tsconfig` `paths` mapping resolves `@mkbabb/glass-ui` → glass-ui `src/index.ts` (a config fact, not a script); `vue-tsc` green with glass-ui's `dist/` deleted is the binding evidence.
- **inv-K-5 — Demo gates green against no live backend.** The demo e2e suite passes with zero console errors when no `api/` backend is reachable: optional reads (color-names) fail-silent-by-design (a designed empty/fallback, not a thrown CORS error), OR the harness intercepts them; no production-origin fetch leaks a console error. (Hardens the I-tranche `ApiProblem` work — the consumer-side failure path was never made console-clean for the optional-read case.) **Structural enforcement**: the existing `zero console errors` Playwright assertions pass with the api/ container down.

**Amendment to inv-I-1 (cross-repo source boundary).** I codified "value.js commits never touch fourier source." K **preserves that for fourier** (untouched) but **opens it for glass-ui**: K is a *paired-authorship* cohort with glass-ui, so K commits MAY touch glass-ui source under the cohort coordination in `coordination/glass-ui.md`. The boundary is repo-specific: closed to fourier, open to glass-ui-the-cohort-peer. This is the structural change that finally unblocks the 7-tranche-carried glass-ui asks.
