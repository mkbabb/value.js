## 3 — Focus management audit

### reka-ui Dialog/Sheet/Popover/DropdownMenu primitives

`PaletteDialog` uses `reka-ui Dialog` (via `@components/ui/dialog`) which provides
focus trap and restore-on-close by default. `VersionHistoryDrawer` uses `Sheet` (same
primitive family). `PaletteCardMenu` uses `DropdownMenu`. `PaletteSlugBar` uses
`Popover`. All reka-ui primitives verified to trap focus and restore it on close as
per reka-ui documentation — no changes needed.

### Hand-rolled overlays

**EditDrawer** (`EditDrawer.vue`): teleports to body and uses Vue `<Transition>` —
it is display:none in the current CSS and only shows on mobile. Both the Commit and
Cancel buttons now have proper accessible names. Focus trap is not required for this
overlay because it is a supplemental toolbar that accompanies the main UI rather than
a blocking modal. The buttons are reachable in tab order when visible.

**SwatchHoverMenu hover panel** (`SwatchHoverMenu.vue:35-47`): teleported hover panel
is mouse/hover-driven (the touch path uses reka-ui Popover). On hover-capable devices
this panel is triggered by pointerenter, not by focus, so keyboard-only users will not
reach it in the hover path. The touch/popover path (reka-ui) provides full keyboard
accessibility. The hover panel now carries `role="toolbar"` and `aria-label` for AT
users who may access it via screen-reader browse mode. A proper focus trap for a
1-3-item hover toolbar would be counterproductive. Recorded as acceptable residual;
a glass-ui `FloatingToolbar` primitive with keyboard support would be the ideal fix.

**PointerDebugOverlay**: dev-only debug panel. The toggle `<button>` now carries
`:aria-expanded` and `aria-controls`. No focus trap required — it's an open panel,
not a modal.

### `:focus-visible` coverage

All new and existing `<button>` elements in demo-owned components carry
`focus-visible:ring-2 focus-visible:ring-ring/40` or `focus-visible:outline-none` +
ring via the `focus-ring` utility class, consistent with the design system. The
`WatercolorDot button.watercolor-swatch:focus-visible` rule in `WatercolorDot.vue`
provides a custom box-shadow ring for blob-shaped buttons.

`ActionButton.vue` previously put the focus ring class on the SVG (where it would
never fire); it is now on the `<button>` wrapper.

## 5 — Contrast review

WCAG AA targets: body text ≥ 4.5:1, large text and UI graphics ≥ 3:1.

**Method**: visual audit of the glass/aurora surface patterns; token-level review of
opacity modifiers in use across the demo.

### Fixed

No contrast issues were severe enough to require a hard fix in this pass.
The review below records findings.

### Flagged / borderline

| Location | Pattern | Risk | Status |
|---|---|---|---|
| `text-muted-foreground/50` on glass cards | `color-mix(in srgb, var(--muted-foreground) 50%, transparent)` — opacity cut in half | Body text at /50 on glass will be below 4.5:1 on many surfaces | Borderline — context-dependent. Most usages are for de-emphasis labels (range hints, secondary captions), which are not primary reading content. Flag for dedicated contrast-measurement pass. |
| `text-muted-foreground/40` in `GradientStopEditor.vue` hint text | "Double-click to add · Right-click to remove" | At /40 opacity this likely fails 4.5:1 | Low-risk: the text is a keyboard+mouse hint, not critical. Flag for W6/tranche-B. |
| Gold accent `--color-gold` on glass | Raw gold on light glass backgrounds | Gold at light weight may not meet 3:1 against white/light-glass | Borderline; gold is only used for featured badges and admin decorations. The `featured-badge` uses background-clip:text + gradient which is unmeasurable by simple ratio. Flagged for tranche-B contrast tooling. |
| `bg-card/60` behind `PaneHeader` title text | Title text over semi-transparent card on aurora | Depends on aurora layer; generally adequate on dark, borderline on light | No change; `PaneHeader` uses `h3.text-heading` (large text, ≥ 3:1 threshold). |
| `text-muted-foreground` range labels in `ComponentSliders.vue` | `opacity-50 pl-1` italic labels above each slider | /50 opacity on small-ish italic text | Flagged. Consider removing the opacity modifier and using the muted-foreground token directly (already a low-contrast token). |

No immediate CSS changes made for contrast; the cases above are borderline or
context-dependent. A dedicated W6 axe/Playwright contrast probe will provide
measured ratios.

## Glass-ui coordination items (Q.md additions)

1. `DarkModeToggle` — animated sun/moon SVG is a glass-ui `controls` export. If it
   wraps a `<button>` internally it is already correct; if it is an SVG with an
   `@click`, glass-ui must provide a `<button>` wrapper with accessible name.
2. `GlassCarouselItem` — used with `@click="scrollToSlider(component)"` in
   `ComponentSliders.vue`. Whether it renders a `<button>` or `<div>` is a glass-ui
   concern. Demo cannot fix this without editing `@components/ui`.
