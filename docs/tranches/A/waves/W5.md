# A.W5 — Accessibility + animation + e2e integrity

**Opens after**: W4 close.
**Agents**: 3 lanes (A accessibility, B animation correctness, C e2e suite integrity). Disjoint; shared tree.
**Hard gate**: the a11y sweep clears its checklist; demo animations render correctly across viewports and honor `prefers-reduced-motion`; `npm run test:e2e` is green against the post-W4 DOM; the dark-mode re-probe is clean.
**Status**: planned.

## Scope

This wave is the augmentation `audit/HARDEN-5` surfaced: the five-angle research wave never covered accessibility, animation correctness, or the e2e suite, and the user's directive explicitly named animations as "broken" and asked for "clear affordances and feedback mechanisms" and validated user/admin flows. W5 opens after W4 so the demo's structure and interactive states have settled — an e2e suite rewritten against a still-moving DOM would churn.

### Lane A — accessibility sweep

1. ARIA — `audit/HARDEN-5` found only 18 of 85 custom components carry any ARIA. Sweep interactive and landmark elements for missing roles, labels, and `aria-*` state. The e2e specs reach controls by `.lucide-*` icon class, which is evidence the controls have no accessible name.
2. SVG-as-button — `audit/HARDEN-5` found this is a class defect, not the one-off `Ad-3` treated in W4. `DarkModeToggle` is a second instance. Sweep for every clickable SVG and route it through a real `<button>` or glass-ui `Button`.
3. Focus management — audit modals, dialogs, drawers, and popovers for focus trap and focus restore on close. Audit `:focus-visible` coverage across interactive elements.
4. Contrast — measure text and UI-graphic contrast against the glass and aurora backgrounds; body text ≥ WCAG AA 4.5:1, large text and UI graphics ≥ 3:1.
5. `prefers-reduced-motion` — `audit/HARDEN-5` found it honored in only 2 demo files. The goo-blob and aurora RAF loops, the scroll-driven animations, and the Vue `<Transition>`s must all respect it.

### Lane B — animation correctness

1. `animations.css` — `audit/HARDEN-5` found a keyframe redefined inside a media query. Audit every global keyframe for redefinition, dead rules, and media-query scoping errors.
2. Scoped `@keyframes` and Vue `<Transition>` usage across the demo — audit for consistency and correctness. `PaneHeader`'s scroll-driven animation depends on `--pane-scroll` declared in `style.css:195`; W2's `style.css` decomposition can silently break it, so W5 verifies `--pane-scroll` still resolves and the scroll animation runs.
3. The goo-blob and aurora RAF loops — verify they pause on `prefers-reduced-motion` and on tab-hidden, and that they do not contend (the dual-WebGL frame-budget concern `audit/HARDEN-5` raised). Capture a frame-budget sample.
4. Verify the animations the user reported "broken" render correctly once W0–W4 have landed — many were collateral of the boot crash, not animation bugs; this lane confirms which were real.

### Lane C — e2e suite integrity

1. `e2e/` holds 16 Playwright specs; `playwright.config.ts:9` boots the default app. `audit/HARDEN-5` found `npm run test:e2e` is red today because the app is crashed — W0 fixes the boot, but the specs reach elements by selectors (`.glass-dock`, `.palette-dialog`, `.fraunces`, `.lucide-*`) that W1's class fixes, W3's restyle, and W4's decomposition will move.
2. Update the e2e specs against the post-W4 DOM and the post-W5-Lane-A accessible selectors — prefer role/label selectors over `.lucide-*` icon-class selectors, which Lane A's ARIA work makes available.
3. Add e2e-green as a standing duty: W5's gate runs the full suite; W6 and W7 re-run it.
4. The user asked for validated user and admin flows. The e2e suite covers the user flow; confirm an admin-flow spec exists (the `AdminPane` path) or add one, and a mobile-flow spec.

### Dark-mode re-probe

Across all three lanes, the Playwright probes run in both light and dark. `audit/HARDEN-5` found no W0–W4 probe specified a color scheme; W5 closes that gap with an explicit dark re-probe of the surfaces W1–W4 touched.
