## Scope

### Lane A — resolution + repo hygiene

1. Delete the hard alias `vite.config.ts:30` (`"@mkbabb/keyframes.js": .../keyframes.js/dist/keyframes.js`). It shadows conditional exports and points at a file keyframes.js deleted. keyframes.js HEAD `8d824ee` already carries the `development` export condition, so once the alias is gone the dev server resolves keyframes.js to `src/` with no further keyframes.js-side work. The Q.W1 dependency the earlier spec named is discharged.
2. Add `resolve.conditions` — **mode-scoped, not global**. Per `audit/HARDEN-2 §3`: Vite auto-injects `development` for `vite serve` and `production` for `vite build`. Adding `development` to `defaultOptions.resolve` would force it into the `production` library build and resolve glass-ui/keyframes.js to their `src/` TypeScript instead of `dist/`. Add `resolve.conditions: ["development", ...]` only to the dev-mode and the `gh-pages`/`hero-lab` demo-build return objects, never `production`. Any per-mode `resolve` object must spread `...defaultOptions.resolve` so the `@src`/`@styles`/`@components` aliases survive.
3. Add `server.fs.allow` covering the sibling package roots (`../glass-ui`, `../keyframes.js`) to the demo-mode return objects, or set `server.fs.strict: false` for those modes. This fixes the `fira-code-latin.woff2` 403 — a Vite dev-server filesystem-access denial for an asset reached through a symlinked sibling outside value.js's root (`audit/HARDEN-2 MISS-3`). The demo's primary mono font is unrenderable without it.
4. Confirm `package.json` carries no other `dist/`-pinned `@mkbabb/*` alias. `audit/HARDEN-2 MISS-4` recorded a clean negative — no other stale glass-ui named import, no other stale reactive-schema literal — so this is a confirmation, not a hunt.
5. Resolve Q-chron-1 — `audit/HARDEN-2 §5` found the "master diverged" premise false: `git log HEAD..master` is empty, `w.w2.1-value-js-prebuild` is four commits ahead of `master` and fast-forwardable. Fast-forward `master` to `HEAD` before W0's first commit so the demo-facing tranche lands on `master`; record the decision in `PROGRESS.md`.
6. Register the `docs/precepts` submodule in value.js's index at SHA `3310a8c` (glass-ui's pin) and track `.gitmodules`. value.js currently pins no precepts SHA (`audit/HARDEN-1 §6`). This is a precondition for A's first commit and brings value.js's precepts checkout current.

### Lane B — Aurora integration boot fix (A-key-2)

1. `App.vue:321-334` — replace the stale `reactive<AuroraConfig>({colorMode, colors, surfaceMode, ...})` with `reactive<AuroraConfig>({ ...DEFAULT_AURORA_CONFIG })` — the **whole** `DEFAULT_AURORA_CONFIG` spread (exported by `@mkbabb/glass-ui/aurora`). 3. Delete the `const { config: auroraConfigResult }` destructure — `UseAuroraReturn` has no `config` member. `provide("auroraConfig", auroraConfig)` with the demo's own reactive object.
4. `AuroraPane.vue` — render the pane in a disabled "under rework" state for W0; its `inject<AuroraConfig>("auroraConfig")!` then receives a live object but its slider table still targets dead fields. The full idiomatic rewrite against the 30-field `AuroraConfig` is W6 scope and must not be attempted under a boot-unbreak wave (`audit/HARDEN-2 §4`).

### Lane C — GooBlob guard + build hygiene (A-key-3, Q-break-3)
