### Lane A — shadow + radius consolidation (Ac-5, Ac-7, Ac-8, Ac-6)

1. The demo overrides `--shadow-card` with an ad-hoc 8px/80% cartoon recipe (`style.css:37-38`), a fourth shadow language. Pick one: if the pop-art look is the demo's identity, override `--shadow-cartoon`/`--shadow-cartoon-hover` to that intensity and point `--shadow-card` at the cartoon token, so cards and explicit `shadow-cartoon` consumers agree.
2. This decision must reconcile with A.W1's Card-surface choice. `audit/HARDEN-3` flagged the collision: W1 set `:shadow="false"` on the 10 wash panes, so for those panes the `--shadow-card` consolidation is a no-op; the consolidation still binds `ColorPicker.vue` (resting plate, shadow kept) and every `shadow-card`/`shadow-cartoon` consumer. The lane proof doc records which surfaces the change actually reaches.
3. Route dialog/popover elevation through `--shadow-modal`/`--shadow-2xl`; replace the hand-rolled `box-shadow` literals (`PaletteDialog`, `SpectrumCanvas`, `ImageEyedropper`) with tokens.
4. Standardize border radius on the semantic aliases — `--radius-card/panel/dialog/input/button/badge`. Collapse the four parallel forms per role.
5. Ac-6 — the `--glass-opacity-subtle: 0.75` override (`style.css:42`) targets a token glass-ui retired in v0.8.0; it is dead. Override the live rung the panes actually want (likely `--glass-opacity-resting`) and delete the stale line. `audit/HARDEN-3` routed this finding here from its prior unassigned state.
6. Pair every shadow/opacity token change with its `.dark` counterpart. `audit/HARDEN-3` flagged that the `.dark` block must carry the consolidated values or dark mode drifts from light.

### Lane B — φ type scale + hierarchy (Ac-1, Ac-9, Ac-10)

1. Map the 91 raw Tailwind text sizes to the glass-ui φ ladder per role — `text-xs`→`text-caption`/`text-micro`, `text-sm`→`text-small`, `text-base`→`text-body`, `text-lg`→`text-prose`/`text-subheading`, `text-2xl/3xl/4xl`→`text-heading`/`text-title`/`text-display`.
2. Repair the flat hierarchy in `ColorNutritionLabel.vue` — every section heading is identical `text-2xl font-normal` over `text-sm` body. Rebuild on the φ heading→emphasis→body cascade.
3. Audit the 27 `font-normal` uses — keep it where the thin-large poster look is wanted (use `text-display-*`), drop it where it merely cancels an unwanted heading weight.

### Lane C — `@apply` consolidation + duplicated-component normalization (Ac-11, Ac-12)
