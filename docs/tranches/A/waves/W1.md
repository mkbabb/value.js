# A.W1 — Card surface + real-bug sweep

**Opens after**: W0 close + glass-ui Q.W2 Lane A (`Card` props fail-explicit).
**Agents**: 3 lanes (A1 wash-pane Card migration, A2 `ColorPicker.vue` resting-plate Card, B undefined-class + radius bugs). Disjoint; shared tree.
**Hard gate**: the 11 Card sites render their intended surface; the 3 undefined classes resolve; `ColorInput` carries input radius; Playwright re-probe (light + dark) shows no black-shadow regression; `vue-tsc` + `npm test` green.
**Status**: planned.

## Scope

This wave clears the bugs where intended styling is silently lost — a stale prop swallowed, a class name that resolves to nothing. `audit/HARDEN-3` found the 11 `<Card variant="pane">` sites are not homogeneous, so the migration splits into two lanes rather than one batched rewrite.

### Lane A1 — wash-pane Card migration (10 sites)

1. Ten of the 11 `<Card variant="pane">` sites are `wash`-tier scroll-pane hosts. Migrate them to `<Card tier="wash" :shadow="false" :grain="false">`. `audit/HARDEN-3` notes the `:grain="false"` that glass-ui Q.W2 Lane B's proposed `tier="wash" :shadow="false"` omits — without it the panes inherit a grain texture they should not carry.
2. `variant` was never a `Card` prop; `Card` swallowed it and fell back to `tier:"resting"` + `shadow:true`, the hard black drop-shadow in `research/screenshots/A-crash-default-partial-render.png`.
3. Depends on glass-ui Q.W2 Lane A landing `Card` props fail-explicit (invariant 31). With it landed, a missed migration site emits a dev warning or a `vue-tsc` error; the migration is verified by the absence of warnings, not by inspection.

### Lane A2 — `ColorPicker.vue` resting-plate Card (1 site)

1. `ColorPicker.vue` is the protagonist plate, not a scroll pane. `audit/HARDEN-3` found that batching it with the wash panes would flatten it into the background. It wants `tier="resting"` with its shadow kept — the migration here is `variant="pane"` → an explicit `tier="resting"`, preserving the elevated surface.
2. The correct tier is a design decision recorded in this lane's proof doc, since A owns the value.js Card write and glass-ui Q.W2 Lane B's hard-coded `tier="wash" :shadow="false"` is wrong for this site (`coordination/Q.md §1`).

### Lane B — undefined-class + radius bugs

1. `font-mono-code` — undefined, used in 14+ files (`research/Ac` Ac-2). Replace with glass-ui `fira-code` or a `text-mono-*` utility. Mono text currently renders as body serif.
2. `text-2xs` — undefined, 9 sites (Ac-3). Replace with `text-micro`.
3. `text-pane-description` — undefined, on the shared `PaneHeader.vue:5` (Ac-4). Replace with `text-caption text-muted-foreground`. One shared component; the fix propagates to every pane.
4. `ColorInput.vue:15` — input field wears `rounded-2xl` (card radius); change to `--radius-input` (Ac-8).
5. Verify the `.fira-code` utility's load path. `audit/HARDEN-3` flagged `utils.css` `.fira-code` as a real W1 hazard — confirm the class resolves to a live `@font-face` family before using it as the `font-mono-code` replacement, so the fix does not swap one undefined class for another.
6. Re-run the audit grep for each undefined class against HEAD before editing (`research/A-challenge.md` C4) — a glass-ui token addition between audit and wave could change a verdict.
