## 0. Verdict in one line

The app's color lives in exactly five places (slider spectra, palette strips, dock glyphs, the
`pastel-rainbow-text` heading, the gold admin register) and **dies everywhere a user acts**: every
copy/save/publish moment is gray, silent, or — in the save case — *actually destroys data
silently*. The Family-B celebratory canon is shipped, tokenized, and consumed by **zero** demo
files (verified: grep across `demo/` → 0). The fix is not scattering color; it is wiring the
already-canonical celebration grammar to the *action moments*, and letting the icon system's
semantic tints (the one axis C9 scored "strength") reach the three surfaces where it currently
goes muted.

## 2. Icon energy — work-orders

The user's directive names the icons as the model and bounds the move: *increase within
proportion*. Proportion = tint lives on glyphs and small accents (dots, badges, rings), never on
washes or backgrounds.

**WO-D7-1 — Semantic accent per view in `VIEW_MAP`.**
`demo/@/composables/viewSchema.ts:69-76` — extend `PaneConfig` with `accent: string` (an OKLCh
literal or token ref per view: gradient→rainbow sweep, extract→camera coral, generate→wand
violet, mix→blend teal, browse→search blue, atmosphere→sky, blob→droplet, admin-\*→gold).
`VIEW_MAP` is the single source (D.W3 Lane D); every consumer inherits in one edit. Consumers:
the dock collapsed pill glyph (`Dock.vue:199`), the view-select trigger glyph
(`DockViewSelect.vue:57-62`), and `PaneHeader.vue` (a 2px accent tick or tinted glyph beside the
pane title — pairs with the W6.B display-rung promotion; one file lifts 14 panes).

**WO-D7-2 — Un-mute the dropdown icon column.**
`DockViewSelect.vue:84` — replace the binary gray-vs-accent with: every glyph carries its
WO-D7-1 semantic tint at reduced chroma (`color-mix(in oklch, <accent> 55%, var(--muted-foreground))`),
full chroma on the active row and on hover. The 2px leading dot (`:78`) takes the same accent
instead of the single `cssColorOpaque`. The dropdown becomes the app's table of contents in
color — the literal "like the icons" move. (U30a's *audacious space-dropdown* is the
U-DROPDOWN lane; this order is scoped to the view select + item tints.)

**WO-D7-3 — Empty states carry the picked color.**
`EmptyState.vue` (5 consumers) — seed with the live accent: a small `WatercolorDot`
(`@mkbabb/glass-ui/watercolor-dot`, already consumed at 9 sites) tinted `cssColorOpaque` above
the message + an optional CTA slot. The degraded product still carries the pop (C9 move 9;
charter W6.B "empty-state CTAs with colorful pops" — this is its file:line anchor).

### 3.4 🔴 P0 found en route — the save flow silently destroys data

`usePaletteActions.ts:59-63`:

```ts
async function onCurrentPaletteSaved(name: string, colors: PaletteColor[]) {
    await ensureUser();                     // ← UNGUARDED network await
    const palette = deps.createPalette(name, colors);  // ← never runs on failure
```

Live reproduction (D7-10 → D7-11 + `.playwright-mcp/console-2026-06-12T18-16-11-745Z.log`):
with the backend unreachable (dev hits `api.color.babb.dev`, CORS-blocked from localhost — the
`VITE_API_URL` hack N.W6.D already kills), tapping save throws an unhandled `TypeError` through
the component event handler (`ensureUser → createSession → fetch`), the **local**
`createPalette` never executes, the editor resets, the count stays 3, and the user's colors are
gone with zero acknowledgment. Contrast `onPublish:35-41`, which try/catches the identical
call. **Fix shape**: the local save must not be gated on a network session at all —
local-first (`createPalette` unconditionally), `ensureUser()` fire-and-forget/deferred to the
publish path. This lands *before* any celebration work; celebrating a failed save would be
grotesque. (Sibling to U9's reset-broken; the U-FIXES lane should co-own.)
