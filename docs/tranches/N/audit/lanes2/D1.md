## 1 — The dual-hero question: RESOLVED — one voice, two channels

The picker has two candidate heroes (U31 numbers, U30b blob) and today they fight — literally
overlap. The answer is **not** to pick one; it is to put them on **different channels along a
diagonal axis**:

- **The number IS the typographic hero** — the foreground voice, the content, the thing you read.
  Bottom-left gravity of the header, on the reading axis.
- **The blob IS the material hero** — the color made flesh, atmosphere not text. Top-right,
  grid-breaking, half-outside the card (the house overlap idiom). It never carries information the
  numbers carry; it IS the color the numbers describe.

Diagonal composition: blob (top-right, breaking the corner) ⤡ space-name + numbers (left stack) ⤡
spectrum ⤡ sliders. No two elements compete on the same channel at the same position. The φ ladder
sets the size relation: blob visible body ≈ card-width/φ² (480/2.618 ≈ **183px**, vs 112px today);
numbers one φ^(1/2) rung above the space name (§2/§3). The mock's proportions confirm both ratios.

## 2 — The hero-number system (U31 + U2): spec

### The register

- **Rung**: `--type-display-3` (φ³ — clamp(2.618rem, 2rem + 3vw, 4.236rem) = 42→68px; **67.8px at
  1440**, ~1.9× today) via glass-ui's `text-hero` utility with `--text-hero-size:
  var(--type-display-3)`. `text-hero` already ships the full numeric register — `font-feature-
  settings: "ss01","tnum","lnum"`, `font-variant-numeric: tabular-nums lining-nums`, leading 0.84
  (glass-ui `typography.css:201-217`). Do NOT hand-roll tnum again.
- **Face**: the display sans (Plus Jakarta) — the mock shows grotesque numbers, and Fraunces-italic
  digits would be wrong for tabular data. (The face actually *loading* is U-TYPE's work-order;
  this spec composes on top of it.)
- **Units** (`%`): 0.35em, baseline-aligned, italic stays (`ColorComponentDisplay.vue:26-28`
  pattern keeps, scaled em-relative so it rides any rung).
- **Separators** (commas): keep — the mock keeps them — but demoted to `opacity-40 font-light` so
  the digits win.

### Non-columnar flow (U2)

One `inline-flex flex-wrap items-baseline` flow; each channel is an **atomic cell**
(`white-space: nowrap` — digits+unit+comma never break internally). Wrapping happens ONLY at cell
boundaries → the current orphaned-comma break (`92%,`╱`88.8, 20`) becomes impossible by
construction. No grid, no columns — prose-flow, exactly the mock.

### The card-lock (the U31 constraint: the card must NOT resize)

Two reservations, both static per color-space:

1. **Inline (ch) reservation** — each channel cell gets `min-inline-size` in `ch` from the
   channel's worst formatted width under `COLOR_SPACE_RANGES` + `DIGITS` (e.g. Lab: L → 4ch
   (`100` + unit em-box), a/b → 6ch (`-125.0`), alpha → 4ch; hex → 9ch mono single-cell). With
   tabular-nums every digit is the same advance width → **value changes cause zero reflow**;
   space changes swap a precomputed reservation table, not a measured one. The table is derivable
   at module scope from `COLOR_SPACE_RANGES` (`src/units/color/constants.ts`) — no runtime
   measurement, no ResizeObserver.
2. **Block (line) reservation** — the numbers block is a fixed **2-line** box at the display-3
   rung (`block-size: calc(2 * 1lh)` with the 0.84 leading ≈ 114px) for EVERY space — 3-channel
   spaces fill one line and breathe; Lab/XYZ negative-heavy states wrap to two. The header never
   changes height when the value, the space, or the sign changes.

### Fluid (the motion half of "fluid, audacious")

Consume glass-ui **`AnimatedDigit`** (`glass-ui/src/components/custom/animated-digit/
AnimatedDigit.vue`) per channel cell: it damps toward the target via `useAnimatedNumber`, ships
the tnum/ss01/lnum register, and publishes `--digit-count` for the width clamp — purpose-built
for exactly this. Slider drags make the hero numbers ROLL, not jump-cut. Two integration notes:
(a) the cells stay `contenteditable` for direct entry (`ColorComponentDisplay.vue:11` keeps —
editing pauses the damp); (b) verify `useAnimatedNumber` gates `prefers-reduced-motion` —
if not, that is a one-line glass-ui ask (inv-N-9 extends to it on adoption).

**Files**: `display/ColorComponentDisplay.vue` (re-author, ~60 LoC today),
`ColorPicker.vue:14-20` (slot unchanged), reservation table beside `colorSpaceInfo.ts`.

## 3 — The audacious space dropdown (U30a)

### Trigger (the "Lab" wordmark)

- **One fluid rung, not a responsive pair**: replace `text-title sm:text-display`
  (`ColorSpaceSelector.vue:17`) with **`text-display-2`** (φ^(5/2) — 33→53px; **53.3px at 1440**).
  The display rungs are already viewport-fluid clamps — the `sm:` pair was contrived AND is a
  measured no-op (32.9px ≠ 41.9px expected; custom-utility cascade order defeats the variant).
  This also lands the mock's hierarchy: numbers (display-3) one √φ step above the name
  (display-2), name above body — three honest rungs.
- Fraunces italic + live `safeAccent` stays (`:16`) — it is the right idea already; U-TYPE's font
  work-order makes it actually paint Fraunces instead of Times.
- **Chevron scales with the type**: `size-[0.5em]` (em-relative), not the fixed 16px — at 53px
  type the current chevron reads as lint.

### Menu (the part that is NOT audacious today)

Today: an unbounded, opaque, all-mono utility list (`fira-code` on the whole group,
`ColorSpaceSelector.vue:22`; 8px presence-dot `:30-33`; clips off-viewport — see
`u-type-02-space-dropdown-open.png`). The redesign — each item becomes a **specimen row** of the
space itself:

| Element | Spec |
|---|---|
| Space name | display face, `--type-heading` rung, NOT mono (kill the group-level `fira-code`; mono was the "utility list" feel) |
| Live conversion | under the name: the CURRENT color converted into that space, `fira-code text-mono-caption opacity-60` — one `colorUnit2(color, space).toFormattedString(DIGITS)` per row; the menu becomes a teaching surface (this app's whole thesis) |
| Swatch | replace the 8px dot with a **`WatercolorDot`** of the current color (house motif; already consumed at `SpectrumCanvas.vue:22-30`) — visible on every row, ring-marked on the active row |
| Material | glass popover at the floating tier, NOT the flat dark wash |
| Bounds | collision-bound + internal scroll — root is U-DROPDOWN's glass-ui `@source` emission fix (U8/U23); this lane only requires it |
| Size parity | trigger-vs-item parity rides `--dropdown-text` per U-DROPDOWN (U7); the display-2 trigger is the sanctioned *display* exception — the wordmark is a hero element, the rows are the control surface |

**Files**: `display/ColorSpaceSelector.vue:13-37`; needs `COLOR_MODEL_KEY` injection for the
per-space conversion (the component already receives `cssColor`).

## 4 — The blob's place (U30b): bigger, absolutely top-right

Today the blob is a **grid child** of the header (`ColorPicker.vue:22` — `col-start-2 col-span-2
row-span-2 justify-self-end`) that lands top-CENTER (measured canvas center-x 452 vs card center
468) and collides with the numbers (`D1-04-home-fresh.png`). The work-order:

1. **Out of the grid, onto the card**: `position: absolute` on the Card (already `relative` via
   `pane-shell`), anchored top-right with **negative insets** ≈ `-1.5rem top / -1.5rem right` so
   the body breaks the card's corner — the grid-breaking overlap idiom, matching how the canvas
   already overflows upward accidentally; make it intentional and symmetric.
2. **Size**: wrapper `w-[11rem]` (176px body ≈ card-width/φ² = 183; canvas halo 282px at the
   internal 160%). Today's `w-[7rem]` (`HeroBlob.vue:8`) reads as a puck, and at 112px the
   satellites are sub-perceptual (U-BLOB's finding compounds this — size is half of why the
   "satellites don't exist").
3. **Footprint reservation**: the header becomes a single left column with
   `padding-right: clamp(7rem, 35%, 11rem)` — numbers and name NEVER run under the blob; the
   2-line number lock (§2) makes the reservation static.
4. **Z-order**: above the card surface, below popovers/dropdown (the menu must never open behind
   the blob).
5. Color/satellite expressivity is U-BLOB's lane (seed near-white + the `uSatColor[]` §8 ask);
   this placement spec is independent and compatible.

**Files**: `ColorPicker.vue:4,22` (header grid simplifies `grid-cols-3` → one column + the
absolute blob sibling), `visual/HeroBlob.vue:8`.

## 5 — The slider region: the veil capsule (U13) + centering by construction (U14)

### U13 — the ellipse hairline veil, restored idiomatically

Provenance recovered: the pre-N.W1 rail was a **vertical expanded `GlassCarousel`**
(`git show 6cfded5:…/ComponentSliders.vue` — `<GlassCarousel orientation="vertical" :expanded>`),
whose pill-capsule glass chrome WAS the "hairline dock/veil ellipse card". N.W1.A rightly killed
the carousel category-error (`role="tablist"` is correct semantics) but threw away the material
with it — the buttons now float naked (`ComponentSliders.vue:9-15`). Restoration WITHOUT
resurrecting the carousel:

- Wrap the rail column in glass-ui's **veil surface** — the `veil-surface` utility / `Card
  surface="veil"` material (glass-ui `cards.css:60-83`: `--veil-bg` quiet-rung glass,
  `--veil-border` the hairline, `--veil-shadow` the rim) with `border-radius: var(--radius-pill)`
  → the dock-like hairline ellipse capsule, self-stretch over the rail's span.
- `--veil-border: 1px solid color-mix(in srgb, var(--foreground) 12%, transparent)` — hairline,
  glassy, never the black line U26 condemns.
- The existing tablist semantics, roving tabindex, and tooltips (`:16-39`) carry over untouched.

### U14 — letters center EXACTLY, by geometry not by nudging

Root cause (concurred with U-CONTROLS, measured here at 9/10/11/12px growing drift): the rail
distributes letters `justify-around` over the column while each slider ROW carries a caption line
above its track (`:49-51`) — letter pitch ≠ row pitch, and the track sits below its row center.
Design fix, two moves:

1. **Delete the in-row range caption.** `(0% - 100%)` duplicates the rail tooltip
   (`:36` already renders `currentColorRanges[component]`) and the slider thumb tooltip; it is
   cramping noise (U32 "less cramped") and the only reason row-center ≠ track-center.
2. **Letters become per-row grid items** (`grid-row: i+1; grid-column: 1; place-self: center`)
   instead of a `justify-around` column — centering is then exact **by construction** for 3, 4,
   or 5 channels, at any track height. The veil capsule (above) spans the rows behind them
   (`grid-row: 1 / -1`, z below the letters).
3. Letters lift one rung: `text-heading` (25.9px) italic display — they are the instrument's
   engraved channel index, currently underweight against a 24px track and unreadable against
   a future 28px one.

### The tracks themselves

Consumption of the glass-ui `Slider` `variant="spectrum"` (+ the `lg` size axis fix) replaces the
raw-reka fork — that whole work-order is U-CONTROLS' §U15/§U28; this lane's composition assumes
it: **28px lg spectrum tracks**, thumb carrying the live color (`thumb live-color`, the N.W6.C
carry), the touch-gate stack (`ComponentSliders.vue:259-344`) deleted in favor of the primitive's.

## 6 — The card and the in-card φ rhythm (the U32 intersection)

Owned globally by the layout lane, but the picker's internal ladder is this lane's to state.
Measured today: header 194 / spectrum 224 / sliders ≈ 210 with uniform `gap-3` — three blocks of
near-equal visual mass = no hierarchy ("contrived, not modern"). The ladder:

| Region | Role | Spec |
|---|---|---|
| Header (name + numbers + blob) | the voice | 2-line lock ≈ 114px numbers + display-2 name; blob breaks the corner |
| Spectrum | the instrument's face | the φ anchor: `lg:h-[16rem]`, the single largest region — it is the *picker* |
| Sliders | the controls | 4-5 × 28px tracks at φ-stepped `gap`(`--space` ladder), rail capsule left |
| Section gaps | rhythm | φ-ladder steps (8/13/21px), NOT the uniform `gap-3` (`ColorPicker.vue:26`) |

Card material: the picker correctly holds `tier="resting"` frost (`ColorPicker.vue:3`) — keep; it
is the 13 OTHER flat panes that must rise to it (C9/N.W6.B), not the picker that must shout.
Cartoon shadow + rounding stay per the depth-grammar lane (U17/U24/U26).

## 7 — Asides surfaced while probing (not this lane's work-orders)

- **Pane-state pollution repro** (U11-adjacent): with stale storage, the desktop home rendered
  docs+docs, then palettes+gradient, surviving URL navigation to `#/?space=…`

  should own "picker = left/first on home, always."

  consumers in DOM) — render-cost + duplicate-landmark a11y smell.
- **CORS noise**: 2 console errors every load (`api.color.babb.dev/colors/approved` rejects the
  `localhost:9000` origin) — dev-origin allowlist, N.W4/W8 wire item.
- `text-display` (bare) IS a live utility (probe 41.888px) — the inv-N-7 phantom sweep need not
  re-flag it; the no-op here is *cascade order of the `sm:` variant*, a different defect class
  worth one line in the abrogation ledger's silencer list.
