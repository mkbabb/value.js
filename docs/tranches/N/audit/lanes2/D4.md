# D4 — The docs panes: φ-ladder rhythm, the definition gap, measure, KaTeX, display ramp

**Lane**: D4 (Fable design lane, second N-fleet, 2026-06-12). Surfaces: `AboutPane.vue` +
`ColorNutritionLabel.vue` + `Markdown.vue`/`Katex.vue` + the 11 `assets/docs/*.md` pages.
**Canon rows**: U4, U5 (LEDGER §G), with U1/U11/U32 cross-refs.
**Method**: live sampling on :9000 (Playwright-driven, own page; the chrome-devtools MCP profile
was lock-held by a sibling). Desktop right-pane is `display:none` at 1440px on the running
server (F1), so the pane was force-shown via an injected `.pane-wrapper{display:block!important}`
style — a measurement workaround only. Spaces sampled live: **Lab** and **OKLCh** (the
`?space=xyz`/`kelvin` URL switches did not take; all 11 docs share one section grammar, so the
findings generalize — verified against `assets/docs/{lab,oklab,oklch,xyz,kelvin}.md` headings).
**Shots**: `shots/D4-01..08-*.png`.

## §1 Findings (each grounded; user rows outrank)

### F1 — Pre-req honesty: the docs pane currently inherits the W2.B emission gap (U11)

At 1440×900 the About pane's wrapper computes `display:none` — `hidden lg:block` with
`lg:block` un-emitted on the running server. Probe-confirmed un-emitted candidates that land
ON the docs surface: `lg:block` (App.vue pane wrappers), `sm:px-6` (AboutPane.vue:20,26 —
content pads 12px on desktop), `p-(--card-spacing)` (ui Card content — **vertical padding
computes 0px**), `font-display` (Markdown.vue:10, ColorNutritionLabel.vue:2). Probes were
exact-element (`getComputedStyle` on synthesized nodes), not selector-grep. The `@source`
directives ARE in the tree (`style.css:60-61`, N.W2.B) — the running dev server predates them
or hasn't re-scanned. **Disposition**: W2.B owns this; every D4 measurement below separates
"dies with W2.B" from "survives W2.B." The killer finding (F2) **survives**.

### F2 — THE HEADLINE: the entire markdown stylesheet is structurally dead (U5 root cause; survives W2.B)

`Markdown.vue:68-316` styles the docs body in a `<style scoped>` block using native CSS
nesting. The Vue scoped compiler appends the scope attribute to **every nested child
selector** — live cssRules read back as `& > h2[data-v-831fb1ef]`, `& ul[data-v-831fb1ef]`,
`& pre[data-v-831fb1ef]`… — but the markdown-rendered children (compiled by
unplugin-vue-markdown into a separate component) **carry no scope attribute** (verified:
`h2Attrs: []`; only the `.markdown-body` root inherits `data-v-831fb1ef`). Every nested rule
is therefore unmatchable. Measured consequences, live (Lab doc, 1440px):

| Rule (Markdown.vue line) | Intent | Live computed |
|---|---|---|
| `> h2` :109 | 30px semibold, `--md-color-h2` | **16px / 400 / foreground** |
| `> h3` :114 | 24px semibold, `--md-color-h3` | **16px / 400** |
| `> p` :140 | `mb-4 leading-7` | **margin 0** |
| `ul/ol/li` :152-166 | `mb-4 pl-8` / `mb-2` | **margin 0** (only browser defaults via Tailwind preflight = 0) |
| `pre` :177 | `p-4 rounded-2xl mb-4 overflow-x-auto` | **padding 0, margin 0** — flush white slab (D4-08) |
| `hr` :228 | `my-4` + h2-tint | **margin 0** |
| `mark.cs-name` :133 | transparent bg, `--md-color-accent` | **raw yellow highlighter** (D4-04: "Lab" in 1976) |
| KaTeX block `> div.inline-block:has(>.katex)` :222 | block + `overflow-x-auto` | **dead → formulas clip** (F6) |
| `> *:not(:first-child)` :77 | `content-visibility:auto` perf | **dead** (the perf optimization never ran) |

The `--md-color-*` producer is alive and excellent (`useMarkdownColors.ts:43-45` — OKLab
safe-accent from the live picker color, hue-rotated h2/h3/accent) with **zero living
consumers**. This is the sibling failure mode to inv-N-7's phantom classes: **phantom
selectors** — rules that exist in the sheet but can never match. u05's "padding inconsistent —
too much or none" is this: *none* inside the markdown (all rhythm rules dead), *too much* at
the one alive seam (AboutPane's own `mb-6`).

### F3 — U4: the definition gap, measured

Header bottom y=223 → Definition alert top y=224: **the entire seam is the 1px Separator**
(AboutPane.vue:18). Contributors: `CardContent` vertical padding 0 (F1, `p-(--card-spacing)`
un-emitted), PaneHeader `pb-2` (8px) inside the sticky region. Post-W2.B the seam becomes a
uniform 24px — better, still flat: the Definition is the **lede** of the page and gets no
typographic or spatial promotion (16px text in a gray `bg-muted/50` wash,
ColorNutritionLabel.vue:3-8).

### F4 — U5: the sectional rhythm, measured (and the missing spacing system)

The "Detailed Guide" seam (D4-04): nutrition content bottom → **0px** → hairline →
**0px** → 33px `text-title` → **24px** (`mb-6`, AboutPane.vue:27) → markdown. The user's
"padding around dividing lines" = both Separators sit flush against content. Inside
ColorNutritionLabel: uniform `gap-4` grid (16px everywhere — no hierarchy between
label↔value, section↔section). **No spacing scale exists anywhere**: zero `--space-*` tokens
in `demo/@/styles/style.css` and zero in glass-ui `tokens.css` (grep-verified). The type
system is φ-laddered (glass-ui typography.css:135-161); the space system is ad-hoc Tailwind
integers. §2.D1 designs the ladder.

### F5 — Measure: 49ch live vs the ~68ch mandate

Markdown body width 454px; ch-probe 9.34px → **49ch** at `--type-body` 16px. The pane is
clamped at `--desktop-pane-max-w: 30rem` (style.css:132) via `lg:max-w-desktop-pane`
(AboutPane.vue:6). 49ch is *below* the 60–75ch comfort band but the real defect is the prose
register (16px body in a reference work that wants `--type-prose` 18px) and the absence of an
element-level cap that would keep prose honest when U32 widens the panes. §2.D5.

### F6 — KaTeX: MathML output at BOTH render sites; matrices mangle; formulas clip (chartered, confirmed live)

`Katex.vue:20` AND `useMarkdownHighlighting.ts:89` render with `output:"mathml"`. Live: Lab
23/23 formulas MathML (0 `.katex-html`), OKLCh 39/39. Evidence: **D4-06** — the
`[l',m',s'] = M₁⁻¹[…]` matrices render as crushed ~10px MathML clusters with miniature
brackets (Chrome MathML Core's matrix layout). **D4-05** — Lab's widest formula is **564px
in a 454px body**: with the `overflow-x-auto` rule dead (F2) and `.about-card`
`overflow-x-hidden` (AboutPane.vue:6), the L*/a*/b* definition line **clips mid-formula**
(`b^* = 200[f(Y/Yₙ` …gone). Code blocks clip identically.

### F7 — The latent font flip: `font-display` on the whole docs body (U1-adjacent; adjudicate BEFORE W2.B lands)

`Markdown.vue:10` (`markdown-wrapper font-display`) and `ColorNutritionLabel.vue:2` put the
`font-display` utility on the **entire** docs body. It is currently un-emitted (F1) so prose
renders Plus Jakarta. The demo's `@theme` maps `--font-display: "Fraunces"` (style.css:64).
**The moment W2.B's emission lands, every paragraph of docs prose silently flips to Fraunces
serif** — an unreviewed whole-register change riding a CSS-infra fix. Decision required now;
the design answer (§2.D2) is: prose stays in the text register; Fraunces is *reserved* for
the display rungs — the editorial serif-display/sans-text pairing the house language wants.

### F8 — Sticky-header bleed-through (new; depth grammar)

D4-06/07: the PaneHeader (`bg-card/60 backdrop-blur-md`, PaneHeader.vue:2) lets the white
`pre` slab and its code text **ghost through** the header while scrolling — "function
oklab2oklch({…" reads superimposed on "About the color spaces,". 60% wash + md blur is below
the masking threshold over high-contrast content. Wants the glass-tier answer, not a magic
number (§2.D3.c).

### F9 — Minor IA truths

- u05's "Color research" is **not an eyebrow** — it is the wrapped tail of the Industries
  list (`colorSpaceInfo.ts:135`) reading as one because the seam above "Detailed Guide" is

## §2 The design answer (work-orders; implementation = N.W6.C)

### D1 — The φ spacing ladder (the U5 ask, made concrete)

Six rungs, geometric on φ around the 1rem body rhythm — mirrors the type ladder's
√φ-geometry so space and type move on one scale:

```css
--space-phi--2: 0.382rem;  /*  6px  φ⁻² intra-cluster: label↔value, eyebrow↔title       */
--space-phi--1: 0.618rem;  /* 10px  φ⁻¹ binding: heading↔its first paragraph            */
--space-phi-0:  1rem;      /* 16px  1   body rhythm: p↔p, li↔li                          */
--space-phi-1:  1.618rem;  /* 26px  φ   sub-section: h3 seam, math-well padding          */
--space-phi-2:  2.618rem;  /* 42px  φ²  section: h2 seam, BOTH sides of every divider    */
--space-phi-3:  4.236rem;  /* 68px  φ³  chapter: header→lede, nutrition→guide            */
```

Composition rules (the rhythm grammar, enforced in docs-prose.css):
1. **Asymmetric binding**: space-above-heading : space-below-heading = **φ : 1**
   (h2: 42px above / 26px below; h3: 26px / 10px). Headings bind to what follows.
2. **Dividers breathe**: every `hr`/`Separator` gets `--space-phi-2` above AND below —
   the literal "padding around dividing lines" row.
3. **Consecutive-heading collapse** keeps Markdown.vue:97's idea: h2+h3 adjacency collapses
   the inner seam to `--space-phi--1`.

**Token home**: glass-ui `tokens.css` §0 + a `@theme` bridge (`--spacing-phi-*` → `mt-phi-2`
utilities) — **a tranche-BA ask** (it serves every consumer; the type ladder already lives
there). Interim: demo-local in `style.css` `@theme` (the D.W4 bridge idiom), migrated at the
3.13.0 pin. KISS: 6 rungs, no half-steps.

### D2 — `docs-prose.css`: kill the scoped block, author the ramp (fixes F2 at the root)

Delete `Markdown.vue:68-316` (the scoped block) and `ColorNutritionLabel.vue:2`/
`Markdown.vue:10`'s blanket `font-display`. Author **`demo/@/components/custom/markdown/
docs-prose.css`** — *unscoped*, imported by `Markdown.vue` (plain `<style>` or a css import),
selectors rooted at `.markdown-body`. No `:deep()` contortion: the stylesheet is a public
contract for foreign-rendered content — scoping was the wrong tool, not a missing attribute.
The mapping (glass-ui ladder, not Tailwind generics — the old rules spoke `text-3xl`):

| Element | Register | Color | Rhythm (D1) |
|---|---|---|---|
| h2 | `--type-heading` (φ), **Fraunces** via `--font-display`, weight 600 | `--md-color-h2` (the live OKLab accent — F2's orphaned producer, reconnected) | phi-2 / phi-1 |
| h3 | `--type-subheading` (√φ), Fraunces 600 | `--md-color-h3` | phi-1 / phi--1 |
| p, li | `--type-prose` (18px floor, fluid), `--font-text`, `text-wrap: pretty` | foreground | phi-0 |
| `mark.cs-name` | inherit + 600 | `--md-color-accent`, transparent bg | — |
| pre | `--type-small` mono, `rounded-panel`, hairline border | glass well (D4 below) | phi-1 pad, phi-2 seam |
| hr | 1px, `--md-color-h2` @ 30% | — | phi-2 both sides |
| table th/td | `--type-small`, tabular-nums | — | phi--1 cells |

Restore `content-visibility:auto` on `.markdown-body > *:not(:first-child)` (the dead perf
rule). **Acceptance gate** (the anti-P9 witness, wired into the W2.B emission probe):
mounted-docs assertion `getComputedStyle(markdown h2).fontSize === --type-heading` — a
phantom-selector regression fails loud, never silently again.

### D3 — AboutPane recomposition: the display ramp + the lede (U4, F3, F9)

a. **The pane h1 goes audacious**: `PaneHeader` title ("About the color spaces, *Lab*") rises
   to `text-display-2` (φ^5/2 fluid) with the **space name in Fraunces italic** carrying the
   `--md-color-accent` tint — the one Fraunces display moment the pane already gestures at
   (the ColorSpaceSelector trigger), promoted to the rung it deserves. The sticky-shrink
   keyframes (PaneHeader.vue:78-85) retarget `--type-display-2 → --type-heading`.
b. **The Definition becomes the lede** (U4): out of the gray `bg-muted/50` Alert
   (ColorNutritionLabel.vue:3) into a glass `tier="frost"` card with hairline + soft cartoon
   shadow; "DEFINITION" as a `text-admin-label` mono eyebrow tinted `--md-color-accent`; the
   definition sentence at `--type-prose` Fraunces-italic — the editorial pull-quote idiom.
   Seam above: `--space-phi-3` (the chapter rung) — the measured 1px dies by design, not by
   padding-accident.
c. **Sticky header = a real glass tier** (F8): replace the ad-hoc `bg-card/60
   backdrop-blur-md` with the glass-ui resting-frost recipe (≥85% wash equivalent + stronger
   blur) so scrolled content can never ghost through. If no exact primitive exists, it is a
   **named tier ask** to glass-ui BA ("sticky pane rail"), not a local opacity bump.
d. **"Detailed Guide" demotes to an editorial divider** (F9): mono-caps eyebrow
   ("DETAILED GUIDE") + extending hairline at `--space-phi-3`, releasing `text-title` so the
   ramp reads pane-h1 → md-h2 → md-h3 without a competing middle title.
e. **Nutrition label**: component ranges + Created/White-Point values go `font-mono
   tabular-nums`; conversion-graph chips (already the pane's best math motif) go mono with
   accent-tinted live-path highlight (the existing `nodeHighlightColor` mechanism, kept).

### D4 — KaTeX: `output:"html"` at both sites + math as exhibits (F6)

- `Katex.vue:20` and `useMarkdownHighlighting.ts:89` → `output: "html"` (the chartered fix;
  katex.min.css + fonts already ship). Kills the D4-06 matrix mangle.
- **Display-math wells**: block formulas sit in a hairline glass well — `rounded-panel`,
  `bg-muted/30`, padding `--space-phi-1`, `overflow-x-auto` restored (un-clips D4-05's
  564px formula), and a **2px left accent rule in the live color** (`--md-color-accent`) —
  the formula becomes a color-coupled exhibit: the math motif singing, within proportion.
  No grid-paper texture (glass-ui paper.css ships grain only; a grid texture is NOT worth a
  new ask — KISS).
- Inline math keeps `mx-1` breathing room (the one rule worth porting verbatim).

### D5 — Measure: cap prose at the element, widen at the layout (F5, with U32)

- `docs-prose.css`: `:is(p, ul, ol, blockquote, dl) { max-inline-size: 68ch; }` — prose
  honors ~68ch at any pane width; tables, wells, code take full width (editorial pattern,
  "handled generally" per U32's clamp doctrine).
- The pane width itself belongs to U32's grand-layout lane: when the dual-pane clamp widens
  (proposal: `clamp(30rem, 44vw, 47rem)` per side at ≥1280px), the docs measure lands
  68ch at `--type-prose` *by the element cap*, not by hand-tuned pane math. At today's
  480px the cap is a no-op — the orders compose, neither blocks the other.
