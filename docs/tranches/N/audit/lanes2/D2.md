### 1.3 WO-E1 [U27, cross-repo] — `EasingConfigurator` as a glass-ui primitive (the port spec)

**The ask (glass-ui-owned, paired-authored; net-new vs tranche BA):** a new subpath `@mkbabb/glass-ui/easing-configurator` exporting `EasingConfigurator` (the EasingEditor contract verbatim), `EasingCurveCanvas`, `EasingSelect`, plus the data module (`EASING_GROUPS`, `getCurvePath`). Specifics:

1. **Contract = the donor's, unchanged** (§1.2 row 1). `readoutValue` stays the complete re-parseable literal (`cubic-bezier(…)`/`steps(…)`), never a bare keyword.
2. **Function source = value.js** — `timingFunctions` + `CSSCubicBezier` via the existing peer (`^0.10 || ^0.11`); the dep edge already exists and stays acyclic (value.js remains the pure sink). The `timingFunctionsAnd` prop bag survives so hosts can extend (keyframes passes its spring-augmented set).
3. **Tokens, not literals**: curve stroke `var(--primary)` with an optional `accent` prop onto the `--section-color-*` ladder; grid/diagonal off `--border`/`--muted-foreground`; the trigger + options on the governed `text-dropdown`/`text-dropdown-secondary` rungs (the donor's J.W7c lesson at `EasingSelect.vue:56-65` — never re-pin option font to a raw caption); `gold-shimmer` for detail curves (already a glass-ui class, V5-verified).
4. **Bounded canvas law**: container-query sizing (`38cqi` clamp 160–280px) + the ±0.6 viewBox overshoot clamp ship as-is — both are donor-proven against pathological hosts.
5. **PRM**: the traveling dot + any preset-change tween gate on `prefers-reduced-motion` (the curve repaint itself is static and exempt).
6. **BA coordination**: glass-ui BA's W-DEMO-AFFORDANCES item (4) already re-conceives **its own demo's** curve picker as a dock-like glass **family chip rack** (12 families, `--glass-bg-floating` selected lift, `FadingScroll`). The primitive should expose the family rack as the configurator's optional group-navigator face (`variant="rack"`) so the BA demo and this primitive are one component, not siblings. Drag-capture: glass-ui inlines its own `useDragCapture` equivalent (the donor's seam is keyframes-local).
7. **U8 inheritance**: the donor's dropdown is the LEDGER's named reference for page-bounded, internally-scrolling dropdown behavior — porting it into glass-ui's Select root discharges the U8 mechanism ask for every consumer at once.

**The consume spec (value.js-owned, the W6-successor wave):** delete `EasingSelector.vue` (66 LoC); the Easing section becomes: per-interval **chip rail** (`1→2`, `2→3`… chips; selected chip = floating-tier lift) + **one** `EasingConfigurator` bound to the selected interval (`update:name → updateInterval(i, name)`, `useGradientModel.ts:155-162`). `resolveEasing` (`useGradientModel.ts:84-88`) stays the fn resolver. `GRADIENT_EASING_NAMES` (24 names, `useGradientModel.ts:57-82`) maps onto the donor families minus Steps (stepped easing is excluded by design for gradients — keep the exclusion, pass a filtered group list).
**Until the glass-ui cut ships**: the interim in-repo lift (WO-E2) must NOT grow a parallel component — it restyles `EasingSelector` in place (tokenized stroke, grouped+thumbed options, larger selected-curve display) and dies on the port.

## §3 — MIX: the gate, the withheld result, preview-honesty (the W6-dead charter items — ALL confirmed live)

### 3.1 The 2.9s gate, measured (D2-12/D2-13; timeline in transcript)

Two colors (#ff2244, #2244ff), Mix clicked at t=0 in an isolated context: `Gathering…` 0–~840ms → `Mixing…` ~840–2390ms → `Revealing…` ~2390–3060ms → **Result appears at t=3061ms**. The result (`oklab(0.5769 0.105 -0.0913 / 1)`) is computed **synchronously at t=0** (`useMixingState.ts:80-82`) and withheld behind a hardcoded `setTimeout` chain (800/2300/2900ms — `useMixingState.ts:84-89,97-102`) + the `v-if="mixResult && animationPhase === 'done'"` gate (`MixPane.vue:109`). Three-plus seconds of spinner for a zero-latency computation, with **no skip affordance** and **no PRM path**.

### 3.2 WO-M1 — Un-withhold the result; the animation becomes celebration, not a gate

1. `startMix` sets `mixResult` immediately and the Result card renders immediately (drop the `=== 'done'` arm at `MixPane.vue:109`); the canvas plays OVER/AROUND the arriving card as a celebratory wash. The spinner row (`MixPane.vue:96-104`) dies — there is nothing to wait for.
2. **PRM**: `prefers-reduced-motion` skips the phase chain outright (`idle → done`, no canvas start) — this also discharges the inv-N-9 LIVE hole: the mix-canvas rAF loop (`useMixingAnimation.ts` — rAF at lines 77/83/99/189/196; zero `matchMedia` anywhere under `mix/`) gains its gate at the composable boundary, one check, not five.
3. **Re-entry guard**: `startMix` only checks `canMix` (`useMixingState.ts:65`) — a second click mid-animation stacks a second timer chain over the first (phases jump backward, timers double-fire). Guard: `if (animationPhase.value !== 'idle' && animationPhase.value !== 'done') return;` + the CTA gets a busy/disabled state for the (shortened) run.
4. The phase timeline compresses: with the result un-withheld, 800/1500/600 → one ~900ms gather-and-suffuse beat is ample (token: `--duration-celebration` family — the zero-consumer Family-B motion tokens the charter names get their first consumer here).

### 3.3 WO-M2 — Preview honesty (quantified)

The mixing-phase suffuse overlay averages **sRGB channel means** (`useMixingAnimation.ts:151-158` and again at `166-181`): for #ff2244 + #2244ff it paints `rgb(145, 51, 162)`, while the actual displayed result is `oklab(0.5769 0.105 -0.0913)` = **`rgb(156, 91, 173)`** (computed against `dist/value.js`) — visibly darker/more saturated than the truth, and the divergence grows with hue distance (sRGB means famously collapse toward mud). Fix at the root: the animation already runs while `mixResult` exists — **pass the real result css into `useMixingAnimation`** (one prop: `resultColor: Ref<string|null>`; `cssToRgb255(resultColor)` replaces both avg blocks). The duplicated avg computation (written twice, lines 151-158 vs 166-169) collapses to one constant. The library that exists to do perceptual mixing must not ship a demo whose mixing preview is non-perceptual.

### 3.4 WO-M3 — The canvas wash discipline

The canvas is `absolute inset-0 … z-controls` over the **entire card including the PaneHeader** (`MixPane.vue:63-66`; D2-12 shows the header drowned in opaque purple at suffuse peak, alpha 0.95). Cap the suffuse alpha ≤ 0.5 over the header band or inset the canvas below the header (`top: var(--pane-header-h)`) — the user must never lose the pane's title to a transient wash. Keep full-bleed for the gather phase (the corner-growth reads beautifully); it is the flat opaque suffuse that violates the glass language.
