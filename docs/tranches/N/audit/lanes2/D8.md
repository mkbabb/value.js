## P0 — D8-1: glass-ui dist ships UNLAYERED utility-named CSS → the demo's entire responsive-variant system is dead (the U11/U32 root cause)

The single highest-leverage finding of this lane. **At 1440×900 the app renders the MOBILE
single-column** (`pane-container--dual` shows only the `lg:hidden` branch at 496px; both
desktop `pane-wrapper hidden lg:flex|lg:block` wrappers compute `display:none`). This is U11
("desktop missing the second right-most pane") and the structural floor under U32's
crampedness — every desktop user gets the phone layout.

**The forensic chain (all live-measured):**

1. The N.W2.B `@source` fix IS in-tree (`fc23c8e`, `demo/@/styles/style.css:60-61`) and the
   emission IS correct: `.lg\:hidden { @media (width >= 64rem) { display: none } }` exists in
   `@layer utilities` (sheet 17, rule-walk position 959), the viewport media matches
   (`innerWidth` 1440, `64rem` ⇒ true).
2. The in-layer `.flex` sits at position 156 — Tailwind's ordering is right; in-layer,
   `lg:hidden` wins.
3. **An UNLAYERED bare `.flex { display:flex }` exists at position 2030, root level of the
   same sheet.** Per css-cascade-5, unlayered author styles beat ALL layered styles regardless
   of order or specificity → it overrides `.lg\:hidden`'s `display:none` on every element
   that stacks `flex` + `lg:hidden` (the mobile branch does exactly that:
   `lg:hidden … flex flex-col …`). Symmetrically the desktop wrappers' `hidden` (layered)
   never gets beaten by `lg:flex` (also layered) — wait, both are layered; the kill shot is
   the unlayered duplicates: `…display:-webkit-box;overflow:hidden}.block{display:block}.flex{display:flex}.grid{…`
   in **glass-ui `dist/styles/components.css`** (glass-ui 3.13.0 local dist; the file —
   and the whole `dist/styles/` tree — contains **zero `@layer`**), imported by
   `@import "@mkbabb/glass-ui/styles"` → `dist/styles/index.css` → `./components.css`
   (`demo/@/styles/style.css:52`). Any utility-named selector glass-ui ships unlayered
   (`.block`, `.flex`, `.grid`, …) silently overrides the demo's layered Tailwind utilities.
4. Effect census: picker = one 478px card in a 1408px main (shot
   `D8-picker-desktop-1440.png` — the dead right half); docs prose measure **42ch** in a
   332px column (target ~68ch, N.W6.C); palettes/gradient render the phone stack
   (`D8-palettes-desktop-1440.png`, `D8-gradient-fullpage.jpeg`).

**Work-orders:**
- **WO-D8-1a (glass-ui, tranche-BA ask — P0):** glass-ui dist CSS must never ship unlayered
  utility-NAMED selectors. Either wrap the compiled output in `@layer glass-ui { … }` at
  build, or namespace the helpers (`.gu-flex`). Cite: web.dev/learn/css/cascade-layers
  ("unlayered styles win over layered styles"); CSS Cascading L5 §6.4. Add a dist gate:
  `grep -E '(^|\})\.(flex|grid|block|hidden)\{' dist/**/*.css` → 0.
- **WO-D8-1b (demo, one-line mitigation — P0, do not wait for BA):**
  `@import "@mkbabb/glass-ui/styles" layer(glass-ui);` (and the `styles.css` twin at
  `style.css:52-53`) — assigning the import to a named layer restores demo-utility
  precedence immediately. Verify glass-ui's internal cascade survives layering (its own
  scoped rules only compete with themselves).
- **WO-D8-1c (CI):** `scripts/css-emission-probe.mjs` proved insufficient — it asserts
  *emission*, and the cascade still killed the utility (the P9 failure mode's second life).
  Extend the boot-smoke with a COMPUTED assertion: mount at 1440×900, assert
  `getComputedStyle(document.querySelector('.pane-wrapper')).display !== 'none'`.

## P0 — D8-2: focus indicators never paint — WCAG 2.4.7 fails app-wide

Real-Tab traversal with `el.matches(':focus-visible') === true` on every probe:

| Control | Computed on focus | Paints? |
|---|---|---|
| `Select color space` trigger (glass-ui DockSelectTrigger) | `outline: oklch(0.62 0.2725 9.8) none 3px`; box-shadow = **five fully-transparent slots** (`rgba(0,0,0,0) 0px 0px 0px 0px` ×5 — the Tailwind ring stack, every slot empty) | **NO** |
| Channel slider thumbs ×4 (reka-ui) | `outline: rgb(28,25,23) none 1px`; shadow `none` | **NO** |
| l/a/b component-value textboxes | `outline: rgb(0,95,204) none 1px` (UA color, style suppressed) | **NO** |
| Dock `Select view` / `Toggle action bar` / `Menu` | outline `none 3px`; transparent shadows | **NO** |
| `channel-rail-item` tab | `outline: solid 2px` | yes — the ONLY painting ring found |

Visual proof: `D8-focus-ring-select.png`, `D8-focus-ring-thumb.png` — focused, ringless.

The tokens exist and resolve: `--focus-ring-shadow` on `:root` computes to a real 2px+8px
ink ring (verified). glass-ui's own idiom is `box-shadow: var(--focus-ring-shadow);
outline:none` (dist CSS, multiple components) — but on the demo's actual controls the var is
never wired in: the demo pattern is Tailwind chains like
`focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring/40`
(`demo/@/components/custom/color-picker/controls/ActionButton.vue:14`) whose assembled ring
slot computes transparent on the glass-ui-styled controls, while `outline-color`/`width` get
set without `outline-style` ever flipping to `solid`.

**WO-D8-2 (demo, root-level — P0):** one `@layer base` rule:
`:where(:focus-visible) { outline: 3px solid var(--ring, oklch(0.62 0.27 9.8)); outline-offset: 2px; }`
then sweep every `focus-visible:outline-none` / `outline:none` (ActionButton.vue:14 et al.)
— each must either consume the glass-ui `--focus-ring-shadow` or fall through to the base
outline; none may suppress with no replacement. Ring contrast ≥3:1 against adjacent pixels
(WCAG 2.4.11 Focus Appearance; web.dev/articles/focus). The audacious accent ring IS the
house language — a colorful pop, purposeful, free.
**WO-D8-2b (glass-ui BA ask):** dock controls (`dock-icon-button`, DockSelectTrigger) must
ship a painting `:focus-visible` ring in dist (their current rules end at transparent rings
under the demo cascade).

## P1 — D8-3: keyboard operability

1. **Gradient stop editor is pointer-only**
   (`demo/@/components/custom/gradient/GradientStopEditor.vue:107-148`): stop handles are
   `<button>`s (named, good) with handlers ONLY `@pointerdown/move/up/cancel` +
   `@contextmenu` (remove) + bar `@dblclick` (add). A focused stop cannot move (WCAG 2.1.1),
   reposition is drag-only (WCAG 2.2 §2.5.7 Dragging Movements — no single-pointer
   alternative), remove is right-click-only (impossible on touch AND keyboard), and the hint
   line documents the exclusions: "Double-click to add · Right-click to remove · Drag to
   reposition" (:147). Handles are `w-5 h-5` = **20×20px** (< 24px, §2.5.8).
   **WO-D8-3a:** per-stop slider semantics (APG slider pattern): Arrow ±1%, Shift+Arrow ±10%,
   Home/End, Delete removes, Enter opens the stop's color editor; visual ≥24px, hit ≥44px.
   This spec should ride INTO the U15/U27 first-class glass-ui slider/easing primitives —
   author it once, in glass-ui.
2. **Dock inert/visibility leak**: `Reset color`/`Copy`/`Random`/`Palettes`/`Extract palette`
   sit inside inert `DIV.dock-layer.dock-layer--full` while rendered at 32×32, y=20,
   opacity 1, `checkVisibility()` true — programmatic focus refused (13 controls in the
   traversal). If that layer is the visible one, pointer users can click what keyboard users
   cannot reach. The layer FSM must couple `inert` + `visibility` + `aria-hidden` atomically
   (glass-ui `useLayerTransition`, the N.W5.D adoption).
   **WO-D8-3b:** atomic state pairing + an e2e assertion: every `cursor=pointer` element with
   a nonzero rect must be programmatically focusable or inside a genuinely hidden layer.
3. **Focus order ≠ reading order** on the picker (WCAG 2.4.3): Tab sequence ran sliders ×4 →
   BODY (a dead Tab stop) → space-select → numeric textboxes — visual order is select →
   numbers → spectrum → sliders. **WO-D8-3c:** reorder DOM to match the visual hierarchy when
   the U32 re-layout lands (one wave, same files).

The repeated mechanism is **opacity/alpha-muting over glass** — and it is also the U1 "gray
wash" in miniature: muting by veil instead of by ink. **WO-D8-6:** mint solid muted-ink
tokens (≥4.5:1 against the glass tiers) and ban `/40`-`/50` alpha text + `opacity:.5` text
app-wide; constrain gradient display-text to stops with ≥3:1 floor (the audacious pops then
read as INK, not haze — the house language wins twice).
*Honesty row:* the segmented-tab "1.4:1" hits are mechanical false positives — white label
over the sibling `segmented-indicator` pill, which an ancestor-walk can't see; visually fine.
