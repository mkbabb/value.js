## 1 — THE ONE HERO PER PANE (the focal architecture)

### The law

1. **Hero-follows-namesake.** Each view (`viewSchema.ts:83-196` `VIEW_MAP`) names one pane (its
   `label`); that pane carries the view's hero. The companion pane **demotes one register**: its
   `PaneHeader` drops one display rung, and its protagonist card de-elevates cartoon → glass-list.
   One viewport, one voice.
2. **One hero per pane, two channels allowed.** A pane may pair ONE typographic hero with ONE
   material hero **on different channels at different positions** (D1 §1's resolution: the picker's
   numbers speak, the blob embodies — diagonal axis, never overlapping). Two same-channel heroes is
   the "contrived" failure.
3. **Everything else is control surface** — body/control registers, quiet materials, zero cartoon.

## 3 — THE DEPTH GRAMMAR (codified so shadow-fighting is impossible)

The house material is **glass plates with cartoon shadows** (U17's verbatim target). Fighting
arises when three depth cues stack on one element inside a pane that has its own (U-cards §0). The
grammar assigns every surface exactly ONE rank; the rank fixes material + edge + shadow + radius:

| Rank | Surface class | Material (glass-ui) | Edge | Shadow at rest → hover | Radius |
|---|---|---|---|---|---|
| **Z0** | the atmosphere | aurora canvas (alive, drifting) | — | — | — |
| **Z1** | pane plates (both panes) | `Card tier="resting"` frost | `--card-edge` | **`--shadow-card` 8px cartoon** → static (plates don't hover) | `rounded-card` |
| **Z1v** | window panes (atmosphere, blob) | `Card surface="veil"` quiet | `--veil-border` hairline | none | `rounded-card` |
| **Z2** | in-pane grouping (slider veil capsule U13, dashed wells, sticky rails) | `veil-surface` / `.dashed-well` | hairline only | **none — depth by material, never shadow** | `rounded-panel` / `radius-pill` |
| **Z3** | the protagonist card (≤1 per pane, per §1 table) | glass plate | `--card-edge` | `shadow-card` 8px → `shadow-card-hover` 10px + `translate(-2px,-2px)` | `rounded-card` |
| **Z3'** | list cards (saved/browse grids, dialog rows) | glass (`bg-card/55` + blur) | `--card-edge` | `shadow-cartoon-sm` → `shadow-card` + lift | `rounded-card` |
| **Z4** | floating chrome (dock, popovers, dropdowns, drawers) | `glass-floating` | glass ring | the floating soft ladder — **NEVER cartoon** | `rounded-panel`/pill |

**The six laws** (normative; write into `demo/DESIGN.md §Depth` as the binding text):

1. **One shadow voice per element** — cartoon offset XOR soft ring, never both (the U24 "extreme"
   = both at once).
2. **The cartoon budget**: per viewport at rest, exactly the Z1 plates + ≤1 Z3 protagonist per
   pane carry the hard offset. A DOM census exceeding `plates + protagonists` is a build failure
   (acceptance gate §6).
3. **Hover lifts, never lurches** — shadows grow one rung from a resting base with a translate
   toward the light; `none → 10px` transitions are banned (D3 §1's recipe is canonical).
4. **Hairlines are glassy**: content surfaces use the **`--card-edge`** mint
   (`color-mix(in oklab, var(--foreground) 12%, transparent)` — ONE mint, owned by U-cards/D3 §3;
   do it once). `--border` keeps structural chrome (dock, inputs) but its dark value
   **retunes off navy** (`style.css:222` `hsl(217.2 32.6% 18%)` → warm `hsl(24 8% 26%)`-class; hue
   217 is the only cool surface in a 24/48-warm house — U26's "black hairline" is literally a
   foreign hue).
5. **Window panes are windows**: where the hero IS the atmosphere/material (atmosphere, blob), the
   plate goes veil — frost would occlude its own hero.
6. **Z belongs to the z-tier tokens; shadows never fake elevation** — a Z4 popover over a Z3
   cartoon card never answers with a bigger offset.

**Application sites**: the 9 `tier="wash" :shadow="false"` pane shells (`PalettesPane.vue:2`,
`BrowsePane.vue:2`, `AdminPane.vue:2`, `AboutPane.vue:3`, `ExtractPane.vue:3`,
`GradientPane.vue:20`, `GeneratePane.vue:32`, `MixPane.vue:61` → Z1 resting+cartoon;
`ConfigSliderPane.vue:100` → Z1v veil); `GradientVisualizer.vue:122` + `PaletteCard.vue:7` per
D2/D3; the 13 raw `rounded-lg/xl/2xl` sites sweep onto role tokens (U-cards §1.C census).

## 4 — THE DESKTOP LAYOUT SYSTEM (U32: the actual grid/clamp math)

### 4.1 The clamp has ONE owner

Today the clamp is owned **three times**: the container formula (`style.css:196`, fixed 1000px),
the per-shell `lg:max-w-desktop-pane` (9 pane files + `ColorPicker.vue:2`'s
`max-w-md sm:max-w-lg lg:max-w-desktop-pane` chain), and `--content-max-h`. The law: **the grid
owns the clamp; pane shells never self-clamp** (`w-full` only; the `max-w-3xl mx-auto` mobile
fallback stays scoped < 1024). Delete the 10 `lg:max-w-desktop-pane` utilities.

### 4.2 The tokens + formula (`style.css:132-133, 189-209` rewrite)

```css
:root {
    --pane-min: 30rem;                              /* 480px — today's size becomes the FLOOR  */
    --pane-max: 44rem;                              /* 704px — the per-card ceiling            */
    --pane-gap: clamp(0.5rem, 1.25vw, 1.618rem);    /* φ-stepped: 8 → 26px                     */
}
.pane-container {
    max-width: min(
        calc(100vw - 2 * var(--app-padding-x)),
        calc(2 * var(--pane-max) + var(--pane-gap)) /* ≈ 1434px — the pathological-wide guard  */
    );
}
.pane-container--dual { grid-template-columns: repeat(2, minmax(0, 1fr)); }  /* equal, always */
```

**The resulting math** (each card; gap at its clamp value):

| Viewport | Container | Each card | Margin/side | vs today |
|---|---|---|---|---|
| 1024 | 992 | **490** | 16 | ≈ parity (the floor) |
| 1280 | 1248 | **616** | 16 | +28% |
| 1440 | 1408 | **695** | 16 | +45% (vs 480 in a 220px-margin float) |
| 1536+ | **1434 (capped)** | **704** | symmetric, grows | the clamp |
| 1920 | 1434 | 704 | 243 | vs 460 dead/side today |
| 2560 | 1434 | 704 | 563 (22%/side) | vs 780 (30%/side) today |

Cards stay **equal** (`1fr 1fr` — the "similar size" invariant, kept), **grow fluidly** 1024→1536,
then **clamp** — "handled generally," no per-width media staircase. The in-card content scales by
construction: the display rungs are vw-clamps (§2), the spectrum/slider regions are `%`-based, and
the φ space ladder (§4.5) is rem-based — a 704px card is the same composition, larger.

### 4.3 The aspect law (the portrait pathology, D6-03)

Dual-pane requires **width AND landscape**:

```css
@media (min-width: 1024px) and (min-aspect-ratio: 1.1) { /* dual grid here */ }
```

Portrait at any width runs the single-slot mobile grammar (one pane + the segmented control —
already the app's good behavior < 1024); `--content-max-h` relaxes on portrait
(`clamp(34rem, 80dvh, 64rem)`) so the lone column may elongate. The existing 21/9 override
(`style.css:154-158`) survives, re-expressed against `--pane-max`.

### 4.4 Vertical placement + the dock

- **Optical lift**: `.app-layout` bottom padding `0.5rem → clamp(0.5rem, 5dvh, 3.5rem)`
  (`style.css:175`) — frees the content from the dead geometric center (D6-03's y=553 marooning;
  free space splits ≈ 1 : φ above:below).
- **The dock grows with the stage** (U32 "dock bigger"): the dock band is 55px at 2400 wide
  (probe). Scope a desktop bump at ≥ 1280: `--dock-padding-y: 1rem` + dock-local
  `--size-icon-btn: 3rem` → band ≈ 67px, proportionate to 700px cards. (Dock motion/sizing
  defects are U-DOCK's lane; this is the static scale only.)
- **U11 closes structurally**: the acceptance gate asserts BOTH `.pane-shell`s render > 0×0 on
  every dual route (the 0×0 second shell in D6-02 is the pane-router/W2.B residual — R1's lane
  owns the mechanism; this lane owns the gate).

### 4.5 The φ space ladder goes app-wide

D4's `--space-phi--2..3` ladder (D4 §2.D1: 6/10/16/26/42/68px) is adopted as the **app's one
spacing system**, not a docs-local fix: pane section gaps (`ColorPicker.vue:26`'s uniform `gap-3`
→ φ steps, D1-7; `GradientVisualizer.vue:118`'s `gap-5`, D2 WO-G2), PaneHeader seams, divider
breathing (U5), `CardContent` padding. Interim home `style.css @theme`; the glass-ui BA
token-mint ask stands (D4 D6). Region masses follow the ladder too: per pane, the hero region is
the φ anchor and control regions step down (D1 §6's picker ladder is the worked example).

## 5 — THE GRAY/DARK ANSWER (U1: the color-temperature + contrast plan)

Four chroma sources, one proportion law (D7's: **tint lives on glyphs, strokes, rings, text, and
small accents — never on washes**):

1. **The atmosphere supplies temperature** (Z0): default `motion: "drifting"` (the U-AURORA flip,
   `panes/keys.ts:28`) + the glass-ui `breathing`-register fix (BA ask); `colorEnergy` 0.55 → ~0.7
   so the field reads on both schemes. Glass plates then have something to refract: set
   `--glass-tint-source: var(--accent-live)` at low strength on Z1 plates (an EXISTING glass-ui
   knob — `tokens/glass.css:128-130` mixes it into every `--glass-bg-*`; zero new surface) so the
   frost itself carries the live temperature instead of gray.
2. **The accent axis gets populated** (the root-2 kill): mint `--accent-live` = the
   contrast-guarded live color (the `safeAccent` mechanism the dock already computes —
   `Dock.vue:99`; library `src/units/color/contrast.ts` safeAccentColor) and **re-point
   `--primary` to it** with a guarded `--primary-foreground`. Today `--primary` ≡ `--foreground`
   (probe §0) — every CTA, focus ring, active state, and curve stroke is ink. After: the whole
   interactive layer speaks the picked color — the derive-from-color house voice (blob, aurora,
   dock glyphs) extended to its last mile. D2's tokenized curve strokes and D7's celebration tints
   inherit it for free.
3. **Semantic view accents** (D7 WO-D7-1/2): the `VIEW_MAP` accent axis tints dock glyphs, pane
   header ticks, section labels — the table of contents in color.
4. **The dark ladder separates** (U-TYPE Part B's flat 6–11% band): retune `style.css:215-225` +
   token overrides to a stepped ladder — `--background` 6% / `--card` 12% / `--muted` 16% /
   `--popover` 20% (≥ 4 L-points between adjacent ranks, hue 24 warm throughout; the navy
   `--border`/`--input` 217-hue overrides retune warm per §3 law 4). Dark cartoon shadows stay
   50/55% (`style.css:218-219` — correct already).

**Light mode** needs only roots 2–3 + the Z1 cartoon plates: the field is already the live color
(`body` bg = `lab(92 88.8 20)`, probe) — D6-01's grayness is the unpainted accent axis + the
shadowless wash plates, not the field.
