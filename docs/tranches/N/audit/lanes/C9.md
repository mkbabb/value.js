## 4. TOP 10 HIGHEST-LEVERAGE DESIGN MOVES (ranked, gestalt not band-aid)

Ranked by (house-language uplift Г— breadth-of-panes-touched Г· cost). Each says the gestalt move,
not a per-instance patch.

1. **[P0] Restore desktop panes via the `@source` + project-`@media` toggle (I-layout 1.1).**
   Without this, no desktop design judgment is possible and the product is broken в‰Ґ1024px. Move the
   load-bearing visibility off droppable Tailwind utilities onto the ONE `@media (min-width:1024px)`
   block the project already owns. Breadth: every pane. *The unblock-everything move.*

2. **[P0] Re-baseline the demo on glass-ui 3.10.1+ (rename carousel/tabs imports) (1.2).** Unblocks
   clean boot в†’ unblocks visual verification of moves 3-10. Breadth: build-wide.

3. **[P1] Wire the aurora to the picked color (C2, I3).** The single biggest *visual* uplift: the
   full-viewport background stops fighting the picker (cyan-vs-magenta) and becomes a derived,
   palette-tracking field. `App.vue:212` в†’ glass-ui `deriveAurora` (shipped in glass-ui в‰Ґ3.2.0; verify
   in 3.10.1). Breadth: the background of **all 14 views, light + dark**. This is the 7+-tranche
   chronic mandate; it is *consumable now*. M.W5 specced it.

4. **[P1] Bring the audacious type voice into the panes (I2).** Promote pane titles from `text-heading`
   (25.9px) to a display rung вЂ” `text-display-1`/`-2` (П†ВІ/П†^2.5, fluid 26в†’53px) for the pane H1, keep
   `text-heading` for sub-sections. The language ships up to 352px and the panes use ~26px. This is
   the cheapest way to make every secondary pane feel like the same audacious product as the picker.
   Breadth: all 14 panes via `PaneHeader.vue:3` (one file).

5. **[P1] Give secondary panes one rung more material (I1).** They need not match the picker's
   `resting`, but `wash`+`shadow=false`+`grain=false` is *too* flat once the aurora bleeds through.
   Move to `quiet` (blur 10px, op .50) OR keep `wash` but restore the cartoon-shadow envelope so the
   panel reads as a distinct plate over the field. One decision in the 9 pane shells. Pairs with move 3
   (a derived aurora makes a slightly-more-opaque pane read intentionally, not muddy).

6. **[P1] Fix the blob footprint contract (C3, I5).** Either extirpate the bespoke `goo-blob` into
   glass-ui (M.W7, gated on a glass-ui cut) or pin its canvas to the picker CardHeader cell so it stops
   being a desktop speck / mobile heading-overrun. Breadth: picker + mix + blob views. K.W3 specced it.

7. **[P1] Resurrect the Atmosphere pane (I4/I10).** Rebuild the slider table against glass-ui's live
   `AuroraConfig` (the W0-deferred work) and delete the false "is live" footer. Turns a dead stub into
   the showcase of the (now-live, move 3) aurora. Breadth: the atmosphere view + truthful copy.

8. **[P2] Make the desktop dock un-collapse on first paint, or render the collapsed pill with more
   icon energy (I9).** The dock is the primary nav and the strongest icon surface; booting it to a
   57px grey-ish pill under the aurora wastes that energy. Breadth: every desktop view's first paint.

9. **[P2] Lift the colorful-pop quotient on listing panes.** Browse/palettes/admin are color-starved
   when the API is down (grey skeletons + cyan aurora). Seed empty/degraded states with the picked
   color (the dock already tints with `cssColorOpaque`) so even the empty product carries the pop.
   Breadth: browse, palettes, 5 admin panes.

10. **[P2] Unify the heading-over-aurora contrast guard.** Where a pane heading sits where the cyan
    aurora bleeds strongest (admin-audit top), the Fraunces serif loses contrast. Once move 3 (derived
    aurora) lands, add a subtle pane-header scrim (`bg-card/60` already exists at `PaneHeader.vue:2` вЂ”
    raise it, or add a 1-stop gradient fade) so the title always wins. Breadth: all paned headers.

> Moves 1-2 are the unblockers; 3-5 are the highest *aesthetic* uplift per file touched (each is
> ~1 file: App.vue, PaneHeader.vue, the 9 pane shells); 6-10 are the chronic-mandate + polish tail.

## 5. PROPOSED STRUCTURE вЂ” the standing per-pane frontend-design audit wave (Fable model)

The user mandated a standing per-pane frontend-design audit wave for the next tranche. Proposed shape,
modeled on per-pane agents + the latest glass-ui as the reference:

### 5.1 Pre-conditions (the wave is worthless until these hold)
- **Gate A:** moves 1-2 landed (desktop panes render; demo boots clean). *Auditing desktop design on a
  `display:none` pane or an un-bootable demo is what wasted the K-era doc-only rounds вЂ” DELTA.md В§"Why
  this mattered".* No per-pane agent runs until a Playwright smoke confirms all 14 views render
  in-viewport at 1440 AND 390.
- **Gate B:** the glass-ui reference is pinned and named (local 3.10.1 / registry 3.12.0 / next 3.13.0).
  Every agent judges *against that exact glass-ui DESIGN.md + token tree*, not against memory.

### 5.2 Topology вЂ” one agent per pane, on the Fable model
14 per-pane agents (picker, palettes, browse, extract, mix, generate, gradient, atmosphere, blob,
admin-users/names/audit/flagged/tags) + 2 cross-cut agents (dock/nav; aurora-background-field) +
1 synthesis agent (this lane's successor). Each per-pane agent:
- Captures its pane at **1440Г—900 @2x AND 390Г—844 @2x, light AND dark** (4 shots), via the
  `createRequire`-playwright harness this lane used (`/tmp/c9-capture.cjs`), saved under a
  per-tranche `audit/visual-evidence-<date>/` (the K corpus convention вЂ” keep it).
- Grades 5 axes on a fixed rubric: **design hierarchy В· spacing rhythm В· typography-scale coherence В·
  color/contrast В· motion** (the brief's scoring axes), each 0-3, with a *named pixel + named
  source line* per score (no ungrounded numbers вЂ” the В§0 discipline).
- Cross-checks every token it uses against the pinned glass-ui token tree (radii, glass tiers,
  durations, type ramp) and flags any demo override that diverges from a house recipe.
- Emits an incongruence list tagged P0/P1/P2 + a gestalt fix (never a band-aid) per finding.

### 5.3 The reference contract (Fable discipline)
- A **single shared rubric doc** (authored W1 of the wave) defines the 5-axis 0-3 scale + the house
  language definition + the pinned glass-ui version, so 14 agents grade comparably.
- A **golden-pane exemplar**: the picker pane is the in-repo north star for "house language done
  right" (Fraunces hero, resting glass, cartoon shadow, tinted icons). Each agent scores its pane's
  *distance from the golden pane*, surfacing the I1/I2 flatness gap automatically.
- **Before/after ПЂ discipline** (from K.W6/M.W9): the wave opens with a BEFORE capture set (the
  binding baseline) and closes by re-capturing + diffing, so the design uplift is *measured*, not asserted.

### 5.4 Synthesis + output
- The synthesis agent (this lane's role) builds the cross-pane incongruence matrix (В§3 shape), ranks
  the top-10 leverage moves (В§4 shape), and proposes the wave DAG. It owns the "suffuse the house
  language MORE, within proportion" call вЂ” the per-pane agents find fractures; synthesis decides which
  fractures are systemic (one fix, N panes) vs local.
- Output home: `docs/tranches/<T>/audit/per-pane/<pane>.md` (14) + `synthesis.md` + the shared rubric.
- **Cadence:** standing = re-run the wave each tranche that touches the demo, re-pinned to the then-current
  glass-ui. The corpus + rubric are durable; only the captures + the glass-ui pin refresh.

### 5.5 Anti-patterns this structure prevents (learned from K)
- **Doc-only audits miss layout P0s** (K's first 3 rounds missed D5/D8 entirely вЂ” only the instrumented
  capture + live CSSOM walk found the desktop blank-pane break). в†’ Pixels are mandatory; Gate A is mandatory.
- **Stale anchors** (M assumed glass-ui 3.3.0; we're at 3.10.1/3.13.0). в†’ Gate B re-pins every run.
- **Ungrounded numbers** (the brief's rule). в†’ Every score cites file:line + a named capture.
