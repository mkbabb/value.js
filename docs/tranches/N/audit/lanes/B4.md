## 2. docs/RELEASE.md

**Cross-check**: The CHANGELOG.md H-close entry (in HEAD) itself says `"NO automation (no .github/workflows/release.yml)"` — that was true at H close (2026-05-26) but K.W2 (commit `c4c5842`, which follows H close in git history) created the file. RELEASE.md was never updated.  
**Correction**: RELEASE.md §1 and §6 must be updated to acknowledge the release.yml exists and auto-publishes on `v*.*.*` tags. §3.6 and §3.5 must be reconciled with the automated flow.

**Correction**: change `node.js.yml` → `ci.yml`.

### 2.3 Stale `proof:*` scripts in CI step list (P0)

**Claim** (`docs/RELEASE.md:47-59`): lists 10+ CI steps including `npm run proof:dts-layout`, `npm run proof:resolution`, `npm run proof:no-deprecated`, `npm run proof:no-ts-ignore`, `npm run proof:as-any-budget`, `npm run proof:as-unknown-as-budget`, `npm run proof:no-deep`, `npm run proof:no-bare-builtins`, `npm run proof:codemod-publication`.  
**Reality** (`package.json`): NONE of these `proof:*` scripts exist in `package.json`. They were present at H close (`074df9c:package.json` confirms them) but were entirely removed by K.W2a (`c4c5842`) when the proof-script idiom was retired as overfit (per MEMORY.md `feedback-proof-idiom-retired`). The actual CI steps in `ci.yml` are: lint → typecheck (both programs via `check-types.mjs`) → build → size-budget → vitest → api tests → `npm pack --dry-run` → bench → playwright.  
**Correction**: replace the `proof:*` block in §2 with the actual `ci.yml` step ladder.

### 2.4 `--legacy-peer-deps` not used in release.yml publish (P1)

### 2.5 `scripts/migrate-*.mjs` not in `files` and do not exist (P1)

**Correction**: remove the `scripts/migrate-*.mjs` item from §3.6 (or restore the file and add to `files` if the codemod is still intended to ship).

### 2.6 References to deleted `CONTRIBUTING.md` and `CHANGELOG.md` (P1)

**Claim** (`docs/RELEASE.md:3,32,84,131,189`): references to `CONTRIBUTING.md` in 5 places.
