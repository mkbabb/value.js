## 3. Root cause: single failure cascade

Every failure shares **one root cause**: the Vite dev server (`npx vite --port 8090`, the `webServer` configured in `playwright.config.ts:43`) fails to start because it cannot resolve a CSS import in glass-ui's published stylesheet.

**Error emitted on every test's WebServer stderr:**
```
Error: Can't resolve '../components/custom/tabs/segmented-tabs.css'
  in '/Users/mkbabb/Programming/glass-ui/dist/styles'
```

The chain:

1. `demo/color-picker/` imports `@mkbabb/glass-ui/styles` → resolves to `/Users/mkbabb/Programming/glass-ui/dist/styles/index.css`
2. `dist/styles/index.css:148` contains `@import "../components/custom/tabs/segmented-tabs.css"`
3. The target `dist/components/custom/tabs/segmented-tabs.css` **does not exist** in the glass-ui 3.10.1 local checkout's `dist/` tree. Only these files exist in that directory:
   - `dist/components/custom/tabs/SegmentedTabs.vue.d.ts`
   - `dist/components/custom/tabs/composables/` (dir)
   - `dist/components/custom/tabs/constants.d.ts`
   - `dist/components/custom/tabs/index.d.ts`
4. The CSS file **does exist** in the glass-ui source at `src/components/custom/tabs/segmented-tabs.css` — it was not copied/emitted into `dist/` by the glass-ui 3.10.1 build.
5. Vite's enhanced-resolve throws on every compilation request that touches `Dock.vue` (which imports `@mkbabb/glass-ui/styles`), surfacing as repeated WebServer errors and a degraded (but running) dev server.
6. Although the Vite server does not hard-crash (it still listens on port 8090 — `reuseExistingServer: !process.env.CI` takes effect), the app renders in a broken state: the page shell never mounts. Every test that asserts `getByRole("main", { name: "Color tool panes" }).toBeVisible()` fails with:
   ```
   Error: expect(locator).toBeVisible() failed
   Locator: getByRole('main', { name: 'Color tool panes' })
   Expected: visible
   Timeout: 8000ms
   Error: element(s) not found
   ```

## 6. Additional glass-ui API skew confirmed (compile-time, not runtime)

Beyond the CSS build defect that blocks E2E, two additional glass-ui API mismatches are present in the demo source (confirmed by `npm run typecheck` being RED):

### 6a. `BouncyTabs` renamed to `SegmentedTabs`

- `demo/@/components/custom/panes/PaneSegmentedControl.vue:18` imports `BouncyTabs` from `@mkbabb/glass-ui/tabs`
- `demo/@/components/custom/mix/MixSourceSelector.vue:4` imports `BouncyTabs` from `@mkbabb/glass-ui/tabs`
- `glass-ui dist/components/custom/tabs/index.d.ts` exports only `SegmentedTabs` (no `BouncyTabs`)

### 6b. `glass-carousel` subpath does not exist; `GlassCarousel`/`GlassCarouselItem` not exported

- `demo/@/components/custom/color-picker/controls/ComponentSliders.vue:117` imports `GlassCarousel, GlassCarouselItem` from `@mkbabb/glass-ui/glass-carousel`
- `glass-ui package.json` exports map has no `"./glass-carousel"` entry (missing subpath)
- The `./carousel` subpath (`dist/carousel.d.ts`) exports: `Carousel`, `CarouselContent`, `CarouselDots`, `CarouselItem`, `CarouselNext`, `CarouselPager`, `CarouselPrevious`, `GlassCarouselPager`, `useCarousel` — no `GlassCarousel` or `GlassCarouselItem`

These two issues cause the 3 demo TypeScript errors referenced in the brief and would also cause runtime errors if the CSS blocker were removed.

## 7. Diagnosis summary

The E2E suite is **fully blocked** by a single glass-ui build artifact defect:

`/Users/mkbabb/Programming/glass-ui/dist/styles/index.css:148` references  
`../components/custom/tabs/segmented-tabs.css` which does not exist in `dist/`.

**Evidence:**
- Source file exists: `/Users/mkbabb/Programming/glass-ui/src/components/custom/tabs/segmented-tabs.css` (present)
- Built file missing: `/Users/mkbabb/Programming/glass-ui/dist/components/custom/tabs/segmented-tabs.css` (absent)
- glass-ui local version: 3.10.1, branch `tranche/AY`
- The `dist/` tree has no CSS file for tabs/segmented-tabs anywhere (confirmed by exhaustive `find`)
- The only CSS file for tabs in `dist/styles/` is `dock-controls/tab-button.css`

**Fix scope (not implemented — read-only audit):** rebuild glass-ui locally (`cd /Users/mkbabb/Programming/glass-ui && npm run build`) so `dist/components/custom/tabs/segmented-tabs.css` is emitted, OR (if the glass-ui build is known-broken on tranche/AY) re-pin to a glass-ui checkout/build that does have the file.
