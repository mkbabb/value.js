## §1 — The ONE residual cohort blocker (the binding constraint for N's demo work)

Everything in C3/C9 traces the demo's un-bootability to a **single upstream glass-ui dist
packaging bug**, verified live:

- `glass-ui/dist/styles/index.css:148` → `@import "../components/custom/tabs/segmented-tabs.css";`
- That file **does not exist in dist** (`find dist -name '*segmented*'` = empty;
  `dist/components/custom/tabs/*.css` = empty). The `BouncyTabs`→`SegmentedTabs` rename (AX.W53)
  renamed the CSS but the dist barrel's `@import` points at a file the build never emits.
- Effect: `@import "@mkbabb/glass-ui/styles"` (demo `style.css:23`) → postcss `ENOENT` → HTTP 500
  → blank `#app`. **The whole demo is un-runnable from a clean CSS pipeline**, on every route.

This is **not a value.js bug** (value.js consumes the broken barrel). It needs a glass-ui
**3.13.0** cut whose `dist/styles/index.css` only imports files it emits. **This single cut
un-blocks the entire N demo arm** (per-pane Fable audit, blob adoption, aurora wiring, dock
e2e). N's demo waves gate on it; N's library waves (WithId, parsing, dispatch, modern-web-lib)
do not.

> **Cohort ask to glass-ui (the ONE binding ask):** cut **3.13.0** with a dist CSS barrel that
> imports only emitted files (the `segmented-tabs.css` emission fix). Secondary cohort asks
> (G3 `TooltipContent variant="mono"`, G4 `DockSelectTrigger clampLabel`) ride the same cut but
> are P1 polish, not blockers (the demo carries Tailwind-override workarounds today, C10 §G).
