## §2 — The NEW finding the charter never saw: 3 RED typecheck errors (glass-ui API skew)

`npm run typecheck` is **RED — 3 value.js errors** (the `check-types.mjs` foreign-filter excludes
glass-ui-internal diagnostics, so these 3 are demo-owned):

```
ComponentSliders.vue(117,50): TS2307 Cannot find module '@mkbabb/glass-ui/glass-carousel'
MixSourceSelector.vue(4,10):  TS2305 '@mkbabb/glass-ui/tabs' has no exported member 'BouncyTabs'
PaneSegmentedControl.vue(18,10): TS2305 '@mkbabb/glass-ui/tabs' has no exported member 'BouncyTabs'
```

Root cause: the local glass-ui checkout (3.10.1) **renamed two surfaces** the demo still imports:
- `./glass-carousel` subpath → `./carousel` (glass-ui `package.json exports` has `./carousel`, no
  `./glass-carousel`). Demo callsite: `ComponentSliders.vue:117` `import { GlassCarousel, GlassCarouselItem } from "@mkbabb/glass-ui/glass-carousel"`.
- `BouncyTabs` export → `SegmentedTabs` (glass-ui src has `components/custom/tabs/SegmentedTabs.vue`;
  no `BouncyTabs`). Demo callsites: `MixSourceSelector.vue:4` + `PaneSegmentedControl.vue:18`
  (+ JSX usages at `MixSourceSelector.vue:100`, `PaneSegmentedControl.vue:6`).

This is a **NEW unilateral wave for N** (chase the glass-ui rename), independent of the cohort —
it's a consumer-side compile break, not a missing producer feature. It also reveals the demo is
NOT pinned to a stable glass-ui (the `file:` symlink tracks whatever the sibling checkout is on),
which is precisely the inv-M-6 hazard M flagged.
