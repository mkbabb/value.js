## §Design-decisions (trade-offs RESOLVED)

- **A/B FIRST is a hard sequencing spine, not a suggestion** (`D6.md §0` roots 1–2 are *prerequisites*;
  `WAVES-2.md §N.W12` header). RESOLVED: the wave does NOT parallelize all five lanes — A (fonts) and
  B (accent/dark) land before C/D/E, because audacious type is invisible in Times and chroma is
  impossible while `--primary` is ink. The cost (less parallelism) is accepted; the alternative
  (ramping the wrong family over an ink layer, then re-doing it) is the heavier path.

- **The font split-brain is cured at the SOURCE token, not the runtime var** (`U-TYPE.md §U1
  ownership (a)/(b)`). RESOLVED: override `--font-stack-display`/`--font-stack-text` (the source the
  glass-ui `@theme inline` bridge inlines into the utility), NOT the dead `--font-display` runtime
  var the demo currently authors. The runtime-var path is a measured no-op (the utility never reads
  it); the source-token path makes the utility resolve Fraunces by construction. Trade-off: this
  couples the demo to glass-ui's `--font-stack-*` token names — accepted, because the alternative
  (`!important` family overrides on every `.font-display` site) is the forbidden workaround.

- **The accent axis rides the shipped library `safeAccentColor`, not a new resolver** (inv-N-3;
  `D6.md §5.2`). RESOLVED: `--accent-live` = `safeAccentColor(picked, bgL, minContrast)` —
  the exact function the dock already computes (`Dock.vue:33`). One color-resolution path. The
  alternative (a CSS-only contrast guard, or a second JS resolver) re-forks the resolver N.W2.C
  collapsed.

- **Container queries (`cqi`) over viewport units for in-card scaling** (D8-8; `D6.md §4.2`).
  RESOLVED: `container-type: inline-size` on `pane-wrapper` + `cqi`-based in-card sizing is
  STRUCTURALLY immune to the D8-1 viewport-variant kill class (the unlayered-utility / viewport-variant
  failures W10.D fixed). The cost (container-query browser support) is a non-issue at the demo's
  evergreen target (modern-web guidance: container queries are baseline). Display rungs are the named
  exception — they stay viewport-fluid `clamp()`s (the ramp is a global type system, not a per-card

  sole clamp. The clamp is owned three times today (container formula + 10 shells + `--content-max-h`);
