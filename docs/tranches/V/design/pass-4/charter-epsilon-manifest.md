## §0 THE OBJECTIVE (NG-3 — stated as a rule, not dressed as a measurement)

The tier of every shared file is decided by a **chosen** objective. The census reports **both**;
neither is "ground truth"; the owner rules the objective (routed under **OF-4-widened**, §3):

- **Objective A — COHESION-MIN-EDGE** (α's default, the RATIFIED lens). A shared file stays
  KERNEL/APP unless **all** its consumers live in ONE feature. The kernel-vs-app tie-break for
  an app+shared file is α's cohesion rule (`charter-alpha-harness.mjs:178`, generalized): a file
  that imports same-directory shared siblings (an intra-coupled cluster HEAD) keeps the cluster
  cohesive at app-root. Minimizes *manufactured cross-tree runtime edges*.
- **Objective B — LITERAL-COLOCATION** (the owner's verbatim "components should be COLOCATED
  with their … composables … recursively … pushed toward the consuming feature"). Push a file
  toward its DOMINANT (plurality) consuming feature even at the cost of an UP edge from the
  other features.

Where A and B **diverge**, the file is a fork row (§3). The demo-wide divergence is **7 of 64**
shared files — the OF-4 fork, enumerated (α surfaced only 1, `useColorPipeline`).

## §4 NG-6 — THE KERNEL→APP-ROOT INVERSION (resolved-or-forked)

The scatter leaves exactly **ONE** kernel→app edge (measured by `placement-census.mjs`, VALIDATION
type-only=1; the other 5 KERNEL color files import only KERNEL siblings):

```
KERNEL keys.ts  ──import type UseColorPipelineReturn──▶  APP useColorPipeline.ts
```

`keys.ts` is KERNEL (29 consumers, 6 features) because it holds `COLOR_MODEL_KEY:
InjectionKey<UseColorPipelineReturn>` — the injection key every feature injects. Its **type**
argument is defined at the pipeline (APP). A KERNEL file depending UP on an APP file is the
structural smell the move creates. Two dispositions (ε recommends **R1**; the owner may ratify R2):

- **R1 (RESOLVE — recommended): hoist the pipeline's return TYPE to the kernel.** Extract
  `UseColorPipelineReturn` (or a `ColorModel` contract interface) into a KERNEL type module
  (`composables/color/color-model.types.ts`, tier KERNEL). `keys.ts` imports the type from a
  KERNEL sibling (kernel→kernel); `useColorPipeline.ts` (APP) imports/implements it UP into the
  kernel (app→kernel, a CLEAN up edge). The inversion is gone. Runtime-neutral (type-only today),
  and it makes the shared contract a first-class kernel artifact — the type-lattice follow-through.
- **R2 (RATIFY): the owner accepts the type-only inversion.** It is runtime-erased (an
  `import type`), disclosed, and single. `placement-census.mjs` VALIDATION reports it as
  `type-only=1` so it can never become a silent runtime edge without the gate flagging it.

Either way it is **not a silent post-scatter surprise** — it is measured, named, and dispositioned.
