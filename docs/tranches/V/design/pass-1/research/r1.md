## GAP (b) — `api/model/lib` vs `routes/service/repository`: RATIFIED VARIANT, measured

**Measured (RAN)**: `node proof-backend-structure.mjs $PWD/api` →
`GREEN (0 violations, 2 warnings)`. The only output: two SOFT warnings
(`palette/service/crud-list.ts` 326 L, `palette/service/crud.ts` 310 L — both under the 500 hard
ceiling, over the 300 soft target). **The `routes/service/repository` boundary produces ZERO
structural violations under glass-ui's own gate.**

**Why it passes (the mechanism)**: value.js splits **domain-FIRST** —
`api/src/modules/{admin,color,meta,palette,session}/` — with `{routes/, service/, repository/,
model.ts, schema.ts, __tests__/}` NESTED inside each domain package. The gate's `checkLayerByType`
(G-BE3, `proof-backend-structure.mjs:203-232`) flags a technical-layer dir ONLY when it is a
**direct child of the source root** (`readdirSync(srcRoot)`, line 208) scattering **≥2 distinct
domains**. value.js's source-root children are `modules/` + `platform/` — neither is a
`LAYER_TYPE_DIR`; the `service/`/`repository/` dirs are per-domain and hold ONE domain each. This
IS the §5.1 grammar realized: "domain-vertical package, colocate handler+model+logic FOR one
domain together" — value.js already does it.

**The divergence is purely VOCABULARY.** Mapping value.js → SPEC by-purpose segments (§0.5.4 /
§5.1): `routes/` ↔ `api/` (handlers, same purpose); `model.ts`+`schema.ts` ↔ `model/`
(**already present at the domain root**); `service/`+`repository/` ↔ the SPEC folds domain-logic +
data-access into `lib/` or directly into the package. So the only literal delta is naming
`routes→api`, `service`/`repository`→(fold into package or `lib/`).

**Is `api/model/lib` STRICTLY better? NO.** Three reasons the L-boundary is a ratifiable variant:
1. **§5.2 is explicit**: "per-language NUMBER, constellation-wide GRAMMAR; **each language binds
   its own idiomatic norms**" (STRUCTURE-SPEC:23, §5.2). The GRAMMAR (domain-vertical,
   infra-ring-legit, no grab-bags, shallow depth, unidirectional) is constellation-wide; the exact
   segment NAMES are the per-repo binding.
2. **§8.8 sets the precedent**: glass-ui itself KEEPS the Vue `composables/` essence-name as "a
   recorded FSD divergence — kept" rather than FSD's canonical name. An idiomatic vocabulary
   variant is sanctioned when the grammar holds.
3. **The measured boundary is clean** (A2: 0 route→repository imports, 0 ad-hoc `c.json({error})`,
   largest file 325 L, none >350). Renaming it to match `api/model/lib` is exactly the portfolio's
   named Family-1 failure mode: "**DAMAGE a clean, measured-passing boundary to satisfy a cosmetic
   vocabulary match.**"

**The one honest hazard for CONSTELLATION-CONFORM**: the harvest-pending **v2 recursive**
resolver (§5.1 FOLD1, `walkDirs` at every level, + FOLD5 pluralize-normalize `repository ≡
repositories`) would **FALSE-POSITIVE** on value.js's `modules/palette/service/` (9 files:
crud, crud-list, flags, forks, oklab, ownership, versions, visibility, votes) — its stem-count
heuristic reads 9 sub-operations of ONE domain as "9 domains scattered." The v1 on-disk gate does
NOT (top-level only), but the SPEC's future v2 does. **This is a gate-calibration gap value.js
EXPOSES** — a second leverage vector: value.js would harden the v2 resolver with a
"within-a-domain-package, a by-purpose sub-dir holds one domain by construction" refinement before
G9 promotes constellation-wide (the SPEC already carries similar owed-hardening caveats for
`providers/` and irregular plurals, §5.1 FOLD4/FOLD5).

**Recommendation (b)**: RATIFY `routes/service/repository` as an accepted per-repo variant
(the §8.8 precedent), OR — if conforming to `api/model/lib` — do it as a clean-break rename with
the explicit note that its ONLY justification is constellation vocabulary uniformity, NOT a
structural defect (the gate says there is none).

## GAP (c) — is there a "library" PROFILE? NO — value.js maps to none; author it (leverage)

**The SPEC's profile axis** (from the section structure): §2 = "recursive component-dir schema
(**frontend**)" — SFC/CVA/scoped-CSS-centric; §4 = "demo/**application** grammar"
(§4-STORYBOOK + §4-PRODUCT router/store app); §5 = "**backend** transposition (language-abstracted)"
with a per-LANGUAGE table (TS-backend / Python / Go / Rust, §5.2). **Scope line (STRUCTURE-SPEC:23)**:
"ONE structural grammar for every repo — glass-ui (**the library**) + sibling demo/consumer apps +
polyglot backends." Crucially, glass-ui-"the-library" is a **Vue COMPONENT library** — its §2
profile is inseparably SFC-shaped (§2.1 `<Name>.vue` root, `variants.ts`, scoped `data-v-` ids).

**value.js `src/` is a pure-TS parsing/color library with ZERO `.vue`** (73 files, 19.8k LoC, A3).
It maps to NONE of the three profiles: not FE-component (no SFC), not demo-app (no router), and
the backend §5.2 table has a "**TypeScript (backend)**" row but no "TypeScript pure-library" row.
The closest existing referent is that §5.2 backend-TS row: "ESM subpath exports; `import type`;
barrels PURE RE-EXPORT-ONLY; declare `sideEffects`" — which value.js already satisfies verbatim
(8 `exports` keys, `verbatimModuleSyntax:true`, array-form `sideEffects`).

**Verdict (c)**: the SPEC does NOT name a pure-library profile. The **profile-agnostic LAW half**
(§0–§6: proportion, recursive colocation, 500-ceiling, pure-re-export barrels, no grab-bags, T2
depth, the 4-node DAG, reject-layer-by-type) DOES apply to value.js unchanged. The **UI-specific
machinery** (SFC/scoped-CSS/CVA, §2.1 non-barrel siblings tied to `.vue`, §2.6, §6 G6) does NOT.
**LEVERAGE pole**: value.js is the constellation's first pure-TS-library adopter → it authors the
"library profile" (§5.2 gets a new row: TS pure-library = backend-TS grammar MINUS the app-edge
`api/routes` layer, PLUS the `subpaths/` publish surface) and feeds it back. This is the charter's
§0.2-named "value.js becomes the proving ground and feeds a library profile back to the constellation."

## The LEAD-vs-FOLLOW src-execution poles (charter-mandated; both stated, MEASURED)

**What glass-ui did**: demo/ restructure LANDED at 5.0.0 (§0.2, verified). The **src/ flatten +
export-reshape was DELIBERATELY DEFERRED** to BI/5.1.0 (portfolio A5.1 #1) — the load-bearing
divergence — for the **atomic-transaction-vs-publish risk**: a mass export-surface reshape "cannot
be both atomic and safely published in one cut." Note the nuance: STRUCTURE-SPEC §7 (line 537)
plans a src *flatten* with "**Zero PUBLIC-EXPORT churn** (subpath surface = `src/*.ts` entry files,
untouched; 0 package.json keys)" — i.e. the component-dir FLATTEN is designed to preserve the
publish surface; the bigger EXPORT DE-INDIRECTION (kill `subpaths/`, expose flat peers) is the part
deferred to 5.1.0.

**value.js's export surface (MEASURED)**: `package.json.exports` = **8 keys**
(`.`, `/color`, `/parsing`, `/math`, `/easing`, `/transform`, `/units`, `/quantize`); `src/subpaths/`
= **7 barrels** (color, easing, math, parsing, quantize, transform, units); **10 internal
`index.ts` barrels** in `src/` (the barrel-pure surface). The 7 subpath barrels are already PURE
(A4 — "the model"); the internal `index.ts` files are the divergence.

- **LEVERAGE**: value.js adopting the SPEC on `src/`-class surfaces **executes the constellation
  standard where glass-ui itself has not** — it becomes the proving ground and feeds back (i) the
  pure-TS **library profile** (gap c), (ii) the **`proof:barrel-pure` gate landed on disk** (glass-ui
  has it only PROPOSED, §6 G8), (iii) a **hardened v2 backend resolver** that no longer
  false-positives per-domain by-purpose sub-dirs (gap b). Three concrete artifacts back to the constellation.
- **HAZARD**: value.js inherits the EXACT risk glass-ui deferred for. Its export surface is 8
  published `exports` keys — a mass reshape (e.g. killing `subpaths/` to expose flat peers, mirroring
  the glass-ui 5.1.0 de-indirection) "cannot be both atomic and safely published in one cut." The
  standard is UNPROVEN on a pure-library/src surface. **Mitigation available**: value.js can take the
  §7 posture (flatten/carve WITH zero-public-export churn — the internal `index.ts` god-barrels carve
  to kind-named siblings + the 14 >500-LoC files drain, all behind the untouched 8-key `exports`
  surface) and DEFER any export-map de-indirection exactly as glass-ui did — leading on the LAW half
  while following on the publish-surface reshape. **Both poles must be scored when this family is ranked.**
