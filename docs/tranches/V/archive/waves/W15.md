# V.W15 — Client Auth, Transport, Drafts, and Operation Truth

**State:** PLANNED · **Meta-tranche:** V.H Hardening · **Depends on:** W6, W11–W14

## Goal

Give the browser one auth owner, one transport contract, distinct device/server objects, and persistent truthful operation state.

**V-A159/V-A162/V-A163 current RED:** omit `Content-Type` on bodyless GETs, strictly decode Problem DTOs without fabricated `about:blank` fallback, and derive color-proposal attribution from the authenticated session rather than a caller field. These are clean-cut transport/auth/provenance obligations, not adapters.

## Scope and file bounds

New generic `demo/platform/{transport,auth,storage}`, palette-owned endpoints/DTOs/cache/composables, IndexedDB device drafts/workspace replicas/command and export outboxes, session and operation-state UI primitives, plus `vite.config.ts` and the ordinary local dev launcher/config needed to expose the API through the same browser origin at relative `/api`. Production edge configuration remains W33. W15 owns unsupported/corrupt-storage detection, preservation/export and reset semantics plus the generic durable prepared-byte record; W22 owns storage recovery's visible composition and W23 owns export serialization/handoff composition. Other visual composition is W22–W24. This palette-client rail depends only on its listed predecessors and neither depends on nor gates W17's sibling/bootstrap migration.

## Completion evidence

- A mutation 401 never clears auth directly. The tab holding the `auth-transition/v1` IDB lease performs a fresh `/sessions/me`, publishes only its current lease epoch, and peers re-derive through BroadcastChannel notification; a different valid same-Principal session's occupied-key 409 likewise never signs the browser out. An older response may not publish reactive state over a newer lease epoch. Separately, V makes no false claim about RFC 6265 delivery order: if an already-emitted stale same-name cookie replaces a newer one, the server rejects it and Account enters one explicit signed-out/recovery state without erasing non-secret operation truth or claiming the newer session survived.
- A user intent durably stores its canonical UUIDv7, canonical non-secret request shell, route, target clocks and state before send and reuses it across retry/reload; no server-only claim ID or authoritative digest is required client-side, and completion compacts to its stable result. Registration/recovery persist the handle and command shell but never a recovery secret or cookie, ask for secret re-entry after reload, and reuse that command ID. For ambiguous rotation retry, the user re-enters retained old+new material; after receipt lookup the no-receipt branch verifies both, while the completed branch ignores old and proves only the receipt-bound new credential. Neither secret persists and rotation never executes twice.
- Logout and self-authority effect responses leave the revoked/stale HttpOnly session cookie untouched, so loss before body observation can replay with that exact cookie while the browser session and fixed server row remain; `/sessions/me` re-derives state but never emits cookie deletion. Rotation handles both boundaries: the marked predecessor replays if replacement headers were lost, while a replacement session bearing the same `issuedByCommandId` settles only that receipt if Set-Cookie applied but the body was lost. No JS-readable/in-memory bearer, `X-Session-Token`, `Authorization`, local/session storage, absolute API origin, CORS credentials or anonymous status branch survives.
- The IDB record is acquired transactionally with ephemeral tab UUIDv7, monotonic epoch, 30-second expiry, five-second renewal and `cookieDrainUntil`. Registration/recovery/rotation hold it through response; after an expired owner, the successor waits through `expiresAt+10s` before another cookie-changing request. That drain is a best-effort concurrency reduction aligned with the edge's origin-settlement deadline, not a browser/network-order proof. A late owner may settle non-secret outbox state but cannot publish auth state for a superseded epoch. Arrival-order probes cover both the ordinary newer-cookie result and the standards-permitted stale-cookie result; the latter must fail closed into visible recovery. BroadcastChannel is a wake-up signal, never authority; server AuthFence decides every authorization overlap.
- `npm run dev:stack` implements `PALETTE-LOCAL-RAIL.md`: real `https://value.localhost:4173`, one gitignored/trusted local PKI, the production NGINX+njs/QuickJS edge artifact, private mTLS Hono, relative `/api`, Secure cookie and exact local Origin. Missing/untrusted/wrong certificates fail with an actionable error; HTTP, ignored TLS, direct API/Vite and unsigned proxy fallbacks are absent.
- DeviceDraft, ServerWorkspace, cached WorkspaceReplica, Handle and Release have distinct types/stores and explicit transitions/UI names. DeviceDraft starts at revision one; each committed display/color/tag mutation advances it once in the same IDB transaction as canonical fixed colors, content digest and the selected-tag projection. A WorkspaceReplica stores one saved `WorkspaceDetail` separately from its dirty overlay, so an export consumer can never infer that overlay has the server revision.
- The generic export store is keyed by `(snapshotDigest,format)` and implements exactly the `value.export-operation/v1` record from `PALETTE-EXPORT-CONTRACT.md`: snapshot bytes commit at `captured` before serialization; `ready` atomically adds exact serialized `Uint8Array`, its SHA-256, MIME and filename; `handoff-initiated` retains those bytes; retryable/terminal codes have the closed field combinations. Reload validates prepared bytes and never asks a current serializer to recreate them. Unknown states, missing prepared fields and digest/MIME/filename disagreement enter the existing corrupt-storage recovery decision, not a compatibility decoder or empty reset.
- Corrupt or unsupported schema-v1 storage produces a visible export/reset recovery choice; no silent empty reset, compatibility decoder, migration shim or dual schema remains.
- 412 presents load remote / duplicate local / discard local and preserves both identities until a choice.
- Pending/committed/conflicted/retryable/terminal states persist on the object; no expiring chip or console line is the sole truth.
- Palette endpoint/DTO/operation state is palette-owned; platform contains only generic transport/auth/storage. Aggregate API barrels, slug migration, wildcard `If-Match`, and remote-as-local cloning are absent.
- The typed client exposes every `PALETTE-FACILITIES.md` BUILD/REPLACE target owned by W15/W22–W24 and no RETIRE spelling. Registration, recovery, rotation, current-session, Workspace, Handle, Release, catalog, social, proposal, policy, Principal/grant, report and audit DTOs preserve their distinct IDs and clocks. Q-AUDIT represents committed-effect rows with no outcome field/filter and represents time as the exact union with neither bound or both canonical `{from,to}` strings; it cannot construct, serialize or retain a cursor for a one-sided interval, and changing either bound clears that cursor before request. No generic palette/admin response or route-string escape hatch survives.
- Public catalog/detail/history/fork reads treat absent, malformed, revoked, expired and stale-policy session state as anonymous; only a valid current session personalizes vote membership. The client does not convert optional-auth invalidity into a public-read failure, and browse remains usable immediately after logout while `/sessions/me` independently reports signed out.
