## Goal

Every one of the 40 cold/returning loads in an `M×H` cell reuses that cell's manifest entry byte-for-byte. The observed target/type/result must equal its frozen target, sequence class and terminal state. Any change to a coordinate, hardware surface, accessible target, start state, input sequence or terminal state invalidates every result, regenerates the 36-entry manifest and resets the entire 1,440-load matrix before measurement; a per-cell patch or selective rerun is forbidden.

W31 derives release families from its own production Vite manifest and observed transitive request graph; W32 is not an input. It writes `docs/tranches/V/audit/pi/w31/activation-record.json` with the built entry document and every transitively referenced script, stylesheet, font, image, worker and renderer asset as `{urlPath,byteLength,sha256,activationVector}`. The vector is the asset's 18-bit cold-activation vector over M00–M17. Assets with an identical vector form one family, ordered by the vector then URL-path ASCII order. The record contains every transitive asset exactly once and no empty family. An all-zero vector is unreachable release weight and is deleted before the record is frozen; a causal product asset requiring an activation outside M00–M17 adds that activation before any measured run, resets the matrix and cannot be silently omitted. Thus shared entry/vendor/font assets, every route chunk, Account/history, all five Admin surfaces, KaTeX, Aurora and Blob are counted even when Vite coalesces or renames files. W32 later verifies this record against the same built bytes and freezes its asset projection; it never retroactively supplies W31's families.

Every M-row crosses both fixed hardware columns: **H-D**, the physical MacBook Pro `Mac17,7` (Apple M5 Max, 18-core CPU, 128 GB) on its 120 Hz built-in display at 1440 CSS px in the in-app Browser; and **H-M**, a physical Pixel 7 (Tensor G2, 8 GB) fixed at 60 Hz and 390 CSS px in stable Chrome. H-M is the named real-device surface unavailable to the in-app Browser; no emulated desktop substitutes for it. The exact OS/browser build, DPR and renderer enter the frozen record. H-D uses fixed 40 ms RTT, 10 Mbit/s down and 2 Mbit/s up; H-M uses fixed 150 ms RTT, 1.6 Mbit/s down and 750 kbit/s up. Each `M×H` cell runs 20 independent cold loads and 20 returning loads. Cold clears HTTP cache and origin storage; returning follows one successful priming visit and retains only that cache plus the canonical persisted seed/mode. Each block contains ten canonical light/warm-seed and ten canonical dark/cool-seed runs in a printed alternating order. Results and p75s remain per coordinate, hardware and cache class; pooling, dropping, substituting or rerunning only outliers is forbidden. If either named device is unavailable, W31 is blocked rather than re-profiled on a friendlier machine.
