## Work — expand each RED into a remediation

1. **Right-size the version rail into the release concept.** Retire the mounted
   `/palettes/:slug/restore`, `/versions`, `/versions/:hash` and revert paths; the immutable-release
   CONCEPT survives right-sized (a published Release is the unit of history), replacing legacy
   versions/hash/revert. Public history is immutable with secret redaction; typed conflicts on
   concurrent mutation. No caller attribution on the release.
2. **Indexed prefix search only.** Delete the `$regex` substring/ranked fallback in
   `queries.ts:97–120`; keep bounded, normalized byte-prefix search over the indexed field.
   Malformed/retired query keys fail explicitly; empty query emits `/palettes`, never `/palettes?`.
3. **Session-derived attribution.** Delete `contributor` from `color/schema.ts` and
   `proposals.ts`; derive the proposer from the authenticated Principal in the request context.
4. **Visibility-filtered provenance.** Public fork/provenance output is minimal and release-scoped;
   private/trashed/purged ancestors collapse to `{kind:'unavailable',ordinal}`; raw documents and
   lineage fields never cross the wire. One active-public Release predicate shared by detail + social ops.
5. **Real readiness.** Replace the Mongo-only `/health` with health/readiness derived from the mounted
   typed routes and the full Problem set (liveness + dependency readiness), not a bare ping.
6. **Generated OpenAPI = mounted routes.** Delete the hand-kept `route-table.ts`; generate the
   OpenAPI 3.1 spec from the mounted typed route registry so it cannot drift. The drift reproduction:
   `/restore` (once retired at step 1) is absent from BOTH the wire and the generated spec, and every
   remaining mounted route appears in the generated spec by construction.
7. **`/sessions/me` recovery + idempotency where honest.** Restore a real `/sessions/me` cookie-
   derived auth recovery (no bearer/local-storage token, no clear-on-any-401). Every mutation carries
   UUIDv7 idempotency and replays the exact status/body/header tuple (incl. ETag/Set-Cookie where
   applicable). Request/response decoders are closed and strict — unknown/duplicate/null/trailing
   fields and malformed Problems fail explicitly (no Zod stripping, no fabricated `about:blank`).
