## §Scope (SYNTHESIS §3 T.W8, transcribed)

1. **Per-surface critique passes** (batches of three — E-6), as a **coherence PRE-FILTER
   only** — the terminal taste authority is OWNER RATIFICATION (plan-audit-2 F18). Each pass:
   **drive the owner-visible surface live** (real GPU where it matters — the O-3 probe class;
   never a static-frame-only judgment), judge against the §2 doctrine (D1–D10) **+ the owner's
   verbatim lines** (mandate §0 + addenda — the pass reads the owner's own words for its
   surface), file remediation rows with anchors. Default surface census (re-derived at
   wave-open from the §1.2 map + the live tree): picker · About · palettes/browse + dialog ·
   extract · mix · generate · gradient · easing · dock + nav + menus · boot/atmosphere/blob ·
   admin. No pass may certify taste — a pass VERDICT is "coherent with the doctrine" or a filed
   row, never "beautiful" (lesson 12).
2. **Remediation lanes** land the filed rows (batches of three; single-writer per surface;
   PP-8 per batch; producer-owned rows route to the letter/inbox, NEVER demo forks). A filed
   row that contradicts a ratified §12 ruling or a §4 spec-retirement is NOT landed — it
   routes to the owner (the retired specs are never restored).
3. **THE TASTE CERTIFICATION PACKAGE** (ONE, tranche-level, assembled by a non-authoring
   agent): annotated frames per surface, both schemes, 3 viewports + the boot screencast;
   **every taste axis presented as a BRACKET with both poles named (the T-26 template: "too
   muted ← target → too strong"; t-prompts-recap F2's law), never a point value**; each axis
   indexed to the owner-verbatim line it answers (E-5 support). The package carries the Q14
   adjudication material (the Lighthouse budget/preset pairing decision is WITH the owner at
   W8/W9).
4. **The full oracle slate + budgets re-run in the same wave**: O-1..O-25 as live (honest
   EXPECTED-REDs carried with their packet cites); the §6.2 budget rows; **a fresh Lighthouse
   delta** appended to the PI-1 per-wave ledger; the O-3 headed-GPU probe re-run (its W8 row).

## §Triumvirate dispatch

Mandatory on:

- a remediation row that contradicts a ratified §12 ruling or a §4 spec-retirement row (HALT —
  owner routing, never re-litigation);
- any pass or lane attempting to certify taste, weaken a gate to green the package, or convert
  a bracket into a point value (a process breach — lessons 12/14);
- a critique pass exposing a defect class the oracle slate cannot see (a NEW oracle mint is a
  bounds expansion — the slate is §6.1's, extended only by triumvirate record);
- **loop halt**: the third critique↔remediation iteration on any one surface halts and routes
  (the residual becomes a package bracket for the owner, not a fourth lane pass).

## §No-workaround prohibitions (binding)

- **Critique is a PRE-FILTER, never a certification** — no agent, Fable or otherwise, certifies
  taste (the terminal authority is the owner; lesson 12).
- **Brackets, never points** — a taste axis presented as a single value is a defective package
  row (t-prompts-recap F2; the T-26 template).
- **No gate weakened to green the package** — reds ride into the package as reds.
- **No restoration of the §4 retired specs** — an overruled spec cited as "best practice" by a
  critique pass is a process breach, not a finding.
- **Producer rows route to packets** — the foreign-tree fence holds through remediation.
